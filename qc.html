STEP 2 / 14

原始數據品質管控與 Trimming

用 FastQC、MultiQC、fastp 把髒污從 FASTQ 裡篩掉，再進入下游分析。

Use FastQC, MultiQC, and fastp to clean noise out of FASTQ before any downstream step.

概覽

為什麼 QC 不只是「跑一下 FastQC」？

原始 FASTQ 經常帶有：定序儀末端品質衰減（後段鹼基 Q 值掉到 20 以下）、adapter 殘留（read 長過插入片段時露出 adapter）、polyG 偽訊號（NovaSeq / NextSeq 雙色化學特有）、過度表達序列（rRNA、PCR duplicate 雛形）、tile 失敗、污染物種等問題。

這些雜訊不處理會讓 alignment 軟錯（soft-clip 暴增）、SNV/Indel false positive 增加、RNA-seq 計數偏誤。QC + Trimming 是性價比最高的清潔步驟——花十分鐘可省下後面幾天的痛苦。

Raw FASTQ commonly suffers from: 3' quality decay (Q drops below 20 in late cycles), adapter readthrough (when insert is shorter than read length), polyG artifacts (specific to NovaSeq/NextSeq 2-color chemistry), overrepresented sequences (rRNA, PCR duplicate seeds), tile failures, and contaminating species.

Left untreated, these inflate soft-clipping in alignment, raise SNV/Indel false-positive rates, and bias RNA-seq counts. QC + Trimming is the single highest-ROI cleanup step — ten minutes saved here prevent days of pain later.

💡
核心原則：不要 over-trim！過嚴會把短 read 砍光，造成 paired 失配與 SV breakpoint reads 流失。Q20 floor + adapter removal 是大多數情境的良好預設。 Core principle: Don't over-trim! Aggressive cutoffs destroy short reads, break pairs, and remove SV breakpoint evidence. Q20 floor + adapter removal is a sensible default for most workflows.
核心指標

一、FastQC 必看的六大模組

📉

Per base sequence quality

箱型圖顯示每個鹼基位置（cycle）的 Phred 分數分佈。正常：前段 Q35+，到 read 尾端逐漸下降。警示：中位數掉到 Q20 以下；尤其 R2 通常比 R1 衰減更快（Illumina paired-end 化學特性）。

Boxplot of Phred score per cycle. Healthy: Q35+ early, gradual decline at 3' end. Warning: median below Q20; especially R2, which decays faster than R1 (chemistry artifact of Illumina paired-end).

🅶

Per base sequence content

每個 cycle 上 A/T/G/C 比例。正常：四條線平行（隨機文庫）。警示：RNA-seq 前 10–12 cycle 常因 random hexamer priming 出現偏誤——可接受但要記住；polyG 尾端則需特別處理。

A/T/G/C proportion per cycle. Healthy: four parallel lines (random library). Warning: RNA-seq often shows bias in cycles 1–12 due to random hexamer priming — acceptable but worth noting; polyG tails need special handling.

📊

Per sequence GC content

所有 read 的 GC% 分佈。正常：單峰且接近物種基因組 GC%（人類 ~41%）。警示：雙峰可能代表物種污染；偏離預期 GC 可能是 amplification bias 或 rRNA 污染。

Distribution of per-read GC%. Healthy: single peak near the species GC% (human ~41%). Warning: bimodal peaks may indicate contamination; deviation from expected GC suggests amplification bias or rRNA contamination.

🪞

Sequence duplication levels

重複出現的 unique sequence 比例。RNA-seq、ChIP-seq 預期較高（高表現基因 / peak 區）；WGS 中若 >30% duplicate 通常代表 PCR over-amplification 或 input DNA 不足。

Fraction of duplicated unique sequences. RNA-seq and ChIP-seq expect higher values (highly expressed genes / peak regions); >30% duplicates in WGS usually means PCR over-amplification or insufficient input DNA.

🧷

Adapter content

cycle 軸上 adapter 序列累積比例。read 長 > insert size 時，3' 端會讀進 adapter；用 Trimmomatic / fastp / Trim Galore 切除。短 insert（cell-free DNA、microRNA）尤其需要。

Cumulative adapter content per cycle. When read length > insert size, the 3' end reads into adapter; remove with Trimmomatic / fastp / Trim Galore. Especially important for short inserts (cfDNA, microRNA).

🔁

Overrepresented sequences

頻率異常高的特定序列。常見來源：未切除的 adapter、rRNA（28S/18S）、PhiX 對照、宿主物種污染。可 BLAST 該序列確認來源。

Specific sequences with abnormally high frequency. Common sources: uncut adapters, rRNA (28S/18S), PhiX spike-in, host contamination. BLAST the sequence to identify the source.

決策引導

四、Trimming 策略決策

🌳 Trimming 模式選擇

Q1:
NovaSeq / NextSeq 雙色化學？→ 是 → 開啟 polyG trimming（fastp 預設 on）。沒做的話 polyG 尾巴會錯誤地比對到 G-rich 區域。
Q2:
短 insert（cell-free DNA、microRNA、circulating tumor DNA）？→ 是 → 一定要 adapter trim，並考慮 paired-end overlap correction（fastp --correction）。
Q3:
RNA-seq 用 Salmon/STAR？→ 是 → Salmon 對輕度 adapter 殘留容忍度高，輕度 trim 即可；STAR 對 adapter 較敏感，建議 trim。
Q4:
變異呼叫 (variant calling)？→ 是 → 不要過嚴！過短的 reads 會降低 mapping quality 而被 GATK 忽略。Q20 floor + adapter + min length 36–50 足夠。
Q5:
ONT / PacBio 長讀長？→ 是 → 不要做 base-level quality trimming！只做 length filtering（filtlong）；長讀長的 Q 分數本質意義不同。
Q1:
NovaSeq / NextSeq 2-color chemistry? → Yes → Enable polyG trimming (fastp default on). Without it, polyG tails misalign to G-rich regions.
Q2:
Short insert (cfDNA, microRNA, ctDNA)? → Yes → Adapter trim is mandatory; consider paired-end overlap correction (fastp --correction).
Q3:
RNA-seq with Salmon/STAR? → Yes → Salmon tolerates mild adapter residue (light trim); STAR is more sensitive (recommend trim).
Q4:
Variant calling? → Yes → Don't over-trim! Reads that become too short lose mapping quality and GATK discards them. Q20 floor + adapter + min length 36–50 is enough.
Q5:
ONT / PacBio long reads? → Yes → Don't do base-level quality trimming! Only length filter (filtlong); long-read Q-scores have different semantics.
⚠️
常見錯誤：① 對 ONT 資料用 Q20 sliding window — 會把 reads 切成碎片；② 對 ChIP-seq 嚴格去重複 — 會誤刪 peak 訊號；③ 直接用 FastQC 預設閾值判斷「過 / 不過」— FastQC 的紅綠燈只是參考！ Common mistakes: ① Q20 sliding window on ONT — shreds reads into fragments; ② Aggressive duplicate removal on ChIP-seq — discards peak signal; ③ Using FastQC default thresholds as pass/fail gates — the traffic lights are just hints!
程式碼

QC + Trimming 實作範例

# FastQC（單樣本 QC 報告）/ FastQC per-sample
fastqc -o qc_out/ -t 8 sample_R1.fastq.gz sample_R2.fastq.gz

# fastp（QC + trim 一步完成）/ fastp all-in-one
fastp \
  -i sample_R1.fastq.gz -I sample_R2.fastq.gz \
  -o trim_R1.fq.gz       -O trim_R2.fq.gz \
  --detect_adapter_for_pe \
  --trim_poly_g \
  --cut_tail --cut_tail_window_size 4 --cut_tail_mean_quality 20 \
  --length_required 36 \
  --json fastp.json --html fastp.html \
  --thread 8

# MultiQC（聚合所有報告）/ MultiQC aggregate
multiqc qc_out/ trim_logs/ -o multiqc_report/

# ONT 長讀長 QC / Long-read QC
NanoPlot --fastq sample.fastq.gz -o nanoplot_out/
filtlong --min_length 1000 --min_mean_q 7 sample.fastq.gz | gzip > filtered.fq.gz
# 自寫滑動視窗 trimmer / Custom sliding-window trimmer
from Bio import SeqIO
import gzip

def sliding_trim(rec, win=4, q=20, min_len=36):
    quals = rec.letter_annotations["phred_quality"]
    cut = len(quals)
    for i in range(len(quals) - win, 0, -1):
        if sum(quals[i:i+win]) / win >= q:
            cut = i + win
            break
    if cut < min_len: return None
    return rec[:cut]

with gzip.open("in.fq.gz", "rt") as fin, \
     gzip.open("out.fq.gz", "wt") as fout:
    for r in SeqIO.parse(fin, "fastq"):
        out = sliding_trim(r)
        if out: SeqIO.write(out, fout, "fastq")
# ShortRead — 自訂 QC 與簡易 trim
library(ShortRead)
fq <- readFastq("sample_R1.fastq.gz")

# 末端品質衰減視覺化 / Visualize 3' decay
qm <- as(quality(fq), "matrix")
plot(colMeans(qm, na.rm=TRUE), type="l",
     xlab="Cycle", ylab="Mean Q", main="Per-cycle quality")
abline(h=20, col="red", lty=2)

# 移除尾端品質 < Q20 的鹼基 / Trim 3' bases below Q20
trimmed <- trimTailw(fq, k=2, a="5", halfwidth=2)  # 5 = Q20 in Phred+33
writeFastq(trimmed, "trimmed.fq.gz")

A. 樣本有 G-rich 區段（如端粒）富集A. The sample is enriched for G-rich regions (e.g., telomeres)
