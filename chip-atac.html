STEP 9 / 14

ChIP-seq 與 ATAC-seq

蛋白質結合與開放染色質——peak calling 與差異 peak 分析。

Protein-DNA binding and open chromatin — peak calling and differential peak analysis.

概覽

兩個方法在解決什麼問題？

ChIP-seq（Chromatin Immunoprecipitation Sequencing）：用抗體把某個蛋白（轉錄因子或 histone modification）抓下來，定序其結合 DNA，回答「這個蛋白在基因組哪裡結合？」

ATAC-seq（Assay for Transposase-Accessible Chromatin）：用 Tn5 transposase 插入「開放」（accessible）染色質區域，定序這些片段，回答「哪些染色質位置是開放（可調控）的？」相對於 DNase-seq 與 FAIRE-seq，ATAC-seq 只需 ~50,000 細胞且步驟簡單，已成事實標準。

兩者的數據結構相似：BAM → peak（讀數富集的 genomic interval）→ peak 註釋 → 比較條件間 peak 差異。

ChIP-seq: pull down a protein (TF or histone mark) with antibody, sequence the bound DNA, answer "where in the genome is this protein bound?"

ATAC-seq: Tn5 transposase inserts into "accessible" chromatin, sequenced fragments mark open regions, answering "which chromatin regions are open (regulatory)?" With ~50,000 cells and simple prep, ATAC-seq has replaced DNase-seq / FAIRE-seq as the standard.

Both yield similar data: BAM → peaks (read-enriched intervals) → peak annotation → differential peaks across conditions.

💡
核心原則：都需要 peak caller，而 MACS3（2024+ 取代 MACS2）是事實標準。ATAC-seq 還需 Tn5 偏移校正（--shift 與 tn5_shift）。 Core principle: Both need a peak caller; MACS3 (replacing MACS2 since 2024) is the de facto standard. ATAC-seq additionally requires Tn5 offset correction (--shift and tn5_shift).

peak 類型

二、Narrow vs Broad Peaks

🎯 Narrow Peaks (sharp, focal)

適用對象：轉錄因子 (TF)、H3K4me3 啟動子標記、CTCF。特徵：幾百 bp 寬、邊界清楚、訊號集中。MACS3：預設模式 --call-summits，pileup 模型偵測。

Examples: transcription factors (TFs), H3K4me3 promoter mark, CTCF. Profile: few hundred bp wide, sharp boundaries, focal signal. MACS3: default mode with --call-summits; pileup model.

🌊 Broad Peaks (diffuse, regional)

適用對象：H3K27me3、H3K9me3、H3K36me3 等廣域組蛋白標記。特徵：數 kb–數百 kb 寬、邊界模糊、訊號分散。MACS3：用 --broad --broad-cutoff 0.1。

Examples: H3K27me3, H3K9me3, H3K36me3 — broad histone marks. Profile: kb–hundreds of kb wide, fuzzy boundaries, diffuse signal. MACS3: use --broad --broad-cutoff 0.1.

⚠️
常見坑：① ATAC-seq 沒做 mtDNA 過濾 → 30–80% reads 浪費在粒線體；② 沒用 Tn5 shift 校正 → peak summit 偏 9 bp；③ ChIP-seq 沒做 input control → peak 大量假陽性；④ 計數 peak 時用 raw counts 而非 normalized → DiffBind 結果失真。 Common pitfalls: ① No mtDNA filter on ATAC → 30–80% reads wasted on mitochondria; ② No Tn5 shift → peak summits off by 9 bp; ③ No ChIP input control → flood of false peaks; ④ Counting peaks with raw counts instead of normalized → DiffBind results distorted.
程式碼

ChIP/ATAC 實作

# --- MACS3 (narrow, ChIP-seq TF) ---
macs3 callpeak \
    -t IP.bam -c input.bam \
    -f BAMPE -g hs \
    -n MyTF \
    --call-summits \
    -q 0.05 \
    --outdir macs3_out/

# --- MACS3 broad (H3K27me3) ---
macs3 callpeak \
    -t H3K27me3.bam -c input.bam \
    -f BAMPE -g hs \
    --broad --broad-cutoff 0.1 \
    -n H3K27me3 \
    --outdir macs3_out/

# --- MACS3 ATAC-seq ---
macs3 callpeak \
    -t atac.dedup.bam \
    -f BAMPE -g hs \
    --nomodel \
    --shift -75 --extsize 150 \
    --call-summits \
    -q 0.01 \
    -n S1 --outdir macs3_atac/
# --- ATAC-seq 完整 QC + peak 流程 ---
# 1) 比對（Bowtie2 用 -X 2000 允許長 insert）
bowtie2 -x ref/hg38 -1 R1.fq.gz -2 R2.fq.gz \
   --very-sensitive -X 2000 -p 16 | \
  samtools sort -@ 8 -o atac.sorted.bam -
samtools index atac.sorted.bam

# 2) 過濾 mtDNA、低 MAPQ、duplicate
samtools view -b -q 30 -F 1804 atac.sorted.bam \
  $(samtools view -H atac.sorted.bam | grep SN:chr | grep -v chrM | awk -F'\t' '{print $2}' | sed 's/SN://') \
  | samtools sort -@ 8 -o atac.filt.bam -
gatk MarkDuplicates -I atac.filt.bam -O atac.dedup.bam -M dup.txt --REMOVE_DUPLICATES true

# 3) Tn5 shift（產生 BED）
bedtools bamtobed -i atac.dedup.bam | awk 'BEGIN{OFS="\t"} {if($6=="+"){$2=$2+4; $3=$3+4} else {$2=$2-5; $3=$3-5}; print}' > atac.shifted.bed

# 4) Fragment size 分佈視覺化（R 套件 ATACseqQC）
# 5) Peak calling: 見 b1 tab
# --- DiffBind 差異 peak 分析 ---
library(DiffBind)
library(DESeq2)

samples <- data.frame(
  SampleID  = c("C1","C2","T1","T2"),
  Condition = c("ctrl","ctrl","trt","trt"),
  Replicate = c(1,2,1,2),
  bamReads  = c("C1.bam","C2.bam","T1.bam","T2.bam"),
  Peaks     = c("C1_peaks.narrowPeak","C2_peaks.narrowPeak",
                "T1_peaks.narrowPeak","T2_peaks.narrowPeak"),
  PeakCaller = "narrow"
)

db <- dba(sampleSheet = samples)
db <- dba.count(db, summits = 250)
db <- dba.normalize(db, method = DBA_DESEQ2)
db <- dba.contrast(db, categories = DBA_CONDITION, reorderMeta = list(Condition="ctrl"))
db <- dba.analyze(db)
res <- dba.report(db, contrast = 1)

# 註釋 peak 與最近基因 / Annotate peaks to nearest gene
library(ChIPseeker); library(TxDb.Hsapiens.UCSC.hg38.knownGene)
txdb  <- TxDb.Hsapiens.UCSC.hg38.knownGene
anno  <- annotatePeak(res, TxDb=txdb, level="gene")
plotAnnoPie(anno)
