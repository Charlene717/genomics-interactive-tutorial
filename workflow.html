Nextflow vs Snakemake, containers, reproducibility, HPC and cloud deployment.

概覽

為什麼需要 workflow 系統？

一條 NGS pipeline 動輒 10–30 個步驟，每步驟跑數小時到數天、依賴特定軟體版本、需要 GB 級記憶體與 HPC 排程。用 shell 腳本 雖然能跑，但失敗時要從頭重跑、無法平行多樣本、不易在新機器/雲端重現。

Workflow 系統解決這些問題：自動依賴管理（DAG）、平行化、失敗 resume、容器化執行、HPC/雲端 portable、版本控制與 audit trail。生物資訊兩大主流是 Nextflow（Groovy DSL，企業/雲端主導；2024 citation share ~43%）與 Snakemake（Python-based，學術社群忠實使用者，17%）。

An NGS pipeline easily has 10–30 steps, each taking hours to days, depending on specific software versions and requiring GB-scale memory + HPC scheduling. Plain shell scripts can run them, but failures require full restart, no parallelism, and reproducibility on new machines/cloud is painful.

Workflow systems solve all this: automatic dependency management (DAG), parallelism, failure resume, containerized execution, HPC/cloud portability, version control, audit trail. The two dominant systems are Nextflow (Groovy DSL, ~43% citation share in 2024, dominant in enterprise/cloud) and Snakemake (Python-based, 17%, loyal academic following).

💡
核心原則：2025 主流推薦——不要從零寫，直接用 nf-core 社群 pipeline。sarek（變異）、rnaseq、chipseq、atacseq、methylseq、ampliseq 都是 community-tested、CI 跑過、雲端 ready。 Core principle: 2025 best practice — don't write from scratch; use the nf-core community pipelines. sarek (variants), rnaseq, chipseq, atacseq, methylseq, ampliseq are all community-tested, CI-validated, and cloud-ready.

主流 pipelines

三、nf-core 主流 pipeline 一覽

🔬 nf-core/sarek (variant calling)

當前版本 3.8.1。支援：germline、tumor-only、tumor/normal paired；SNV/Indel (GATK/DeepVariant/Strelka2)、SV (Manta/Delly/TIDDIT)、CNV (CNVkit/AscatNGS)、MSI、HLA typing。適用：WGS/WES/panel-seq 變異呼叫端對端流程。

Current 3.8.1. Supports: germline, tumor-only, tumor/normal paired; SNV/Indel (GATK/DeepVariant/Strelka2), SV (Manta/Delly/TIDDIT), CNV (CNVkit/AscatNGS), MSI, HLA typing. Use: end-to-end WGS/WES/panel variant calling.

📈 nf-core/rnaseq

RNA-seq alignment + quantification + QC。支援：STAR/HISAT2 + Salmon、featureCounts/RSEM；輸出 gene/transcript counts、TPM、MultiQC report；可直接餵 DESeq2。

RNA-seq alignment + quantification + QC. Supports: STAR/HISAT2 + Salmon, featureCounts/RSEM; outputs gene/transcript counts, TPM, MultiQC report; ready to feed DESeq2.

🧶 nf-core/chipseq & atacseq

從 FASTQ 到 peak。包含：BWA/Bowtie2 比對、MarkDuplicates、MACS3 peak calling、HOMER motif、differential binding (DiffBind)、IGV track、MultiQC。

FASTQ to peaks. Includes: BWA/Bowtie2 alignment, MarkDuplicates, MACS3 peaks, HOMER motifs, differential binding (DiffBind), IGV tracks, MultiQC.

🧬 nf-core/methylseq

WGBS / RRBS / EM-seq pipeline。支援：Bismark 或 bwa-meth 比對、CpG methylation extraction、QC（BSseq concordance）、MethylDackel 或 methylKit downstream。

WGBS / RRBS / EM-seq pipeline. Supports: Bismark or bwa-meth alignment, CpG extraction, BSseq concordance QC, MethylDackel or methylKit downstream.

可重現性

四、可重現性的四大支柱

1. 容器化 (Docker / Singularity / Apptainer)

把工具與其完整環境打包進不可變的 image。HPC 環境通常用 Singularity / Apptainer（無 root）；Docker 用於本地/雲端。每個 nf-core process 都對應特定 BioContainer image。

Package tools + full environment into immutable images. HPC commonly uses Singularity / Apptainer (rootless); Docker for local/cloud. Each nf-core process maps to a specific BioContainer image.

2. 版本鎖定

workflow code 用 git tag (-r 3.14.0)；container image digest 鎖定（@sha256:...）；reference genome 標明 source + checksum。

Pin workflow code with git tags (-r 3.14.0); pin container image digests (@sha256:...); reference genome source + checksum.

3. Reference data 標準化

使用 iGenomes / GENCODE 標準 reference；輸入 samplesheet 對應 SRA / ENA accession 而非本地 file。

Use iGenomes / GENCODE standard references; samplesheets reference SRA / ENA accessions rather than local files.

4. 記錄與 audit trail

Nextflow 自動產 .nextflow.log、trace.txt、report.html，記錄每個 process 的執行時間、記憶體、退出碼。發表時建議附這些到 supplementary。

Nextflow auto-generates .nextflow.log, trace.txt, report.html recording per-process runtime, memory, exit codes. Recommended for paper supplementary materials.

Q5:
大型樣本與 cost 敏感？→ 是 → 用 AWS Batch + Spot 或 FSx Lustre 大幅降低 I/O 成本；nf-core/sarek 1000 樣本 WGS 在 spot 約 $40/sample

Quick single-sample test? → Yes → Local Docker / Conda (-profile docker)

實作範例

// --- Nextflow DSL2: 簡易 NGS pipeline ---
nextflow.enable.dsl = 2

process FASTP {
  tag "$meta.id"
  conda "bioconda::fastp=0.23.4"
  container "quay.io/biocontainers/fastp:0.23.4--h5f740d0_0"
  input:  tuple val(meta), path(reads)
  output: tuple val(meta), path("*_trim_*.fq.gz"), emit: trimmed
          path "*.json", emit: json
  script:
  """
  fastp -i ${reads[0]} -I ${reads[1]} \\
        -o ${meta.id}_trim_R1.fq.gz -O ${meta.id}_trim_R2.fq.gz \\
        -j ${meta.id}.json -h ${meta.id}.html --thread ${task.cpus}
  """
}

process BWAMEM2 {
  tag "$meta.id"
  cpus 16; memory '32 GB'
  input:  tuple val(meta), path(reads); path index
  output: tuple val(meta), path("*.sorted.bam"), path("*.bai")
  script:
  """
  bwa-mem2 mem -t ${task.cpus} ${index}/hg38 ${reads[0]} ${reads[1]} \\
    | samtools sort -@ 4 -o ${meta.id}.sorted.bam -
  samtools index ${meta.id}.sorted.bam
  """
}

workflow {
  reads_ch = Channel.fromFilePairs("data/*_{R1,R2}.fq.gz")
            .map { id, files -> [[id: id], files] }
  FASTP(reads_ch)
  BWAMEM2(FASTP.out.trimmed, file("ref/bwa_idx"))
}
# --- Snakemake: 同樣的 pipeline ---
SAMPLES = ["S1", "S2", "S3"]

rule all:
    input: expand("bam/{sample}.sorted.bam.bai", sample=SAMPLES)

rule fastp:
    input:  r1 = "data/{sample}_R1.fq.gz",
            r2 = "data/{sample}_R2.fq.gz"
    output: r1 = "trim/{sample}_R1.fq.gz",
            r2 = "trim/{sample}_R2.fq.gz",
            json = "qc/{sample}.json"
    threads: 8
    container: "docker://quay.io/biocontainers/fastp:0.23.4--h5f740d0_0"
    shell:  """
            fastp -i {input.r1} -I {input.r2} \\
                  -o {output.r1} -O {output.r2} \\
                  -j {output.json} --thread {threads}
            """

rule bwa_align:
    input:  r1="trim/{sample}_R1.fq.gz", r2="trim/{sample}_R2.fq.gz",
            idx="ref/bwa_idx/hg38"
    output: bam="bam/{sample}.sorted.bam", bai="bam/{sample}.sorted.bam.bai"
    threads: 16
    container: "docker://quay.io/biocontainers/bwa-mem2:2.2.1"
    shell:  """
            bwa-mem2 mem -t {threads} {input.idx} {input.r1} {input.r2} \\
              | samtools sort -@ 4 -o {output.bam} -
            samtools index {output.bam}
            """

# 跑：snakemake --use-singularity --cores 32 --resources mem_mb=64000
# --- nf-core/sarek 一行跑完 WGS variant calling ---
nextflow run nf-core/sarek -r 3.8.1 \
   --input samplesheet.csv \
   --outdir results/ \
   --genome GATK.GRCh38 \
   --tools haplotypecaller,strelka,manta,deepvariant \
   --joint_germline \
   -profile docker

# samplesheet.csv 範例:
# patient,sample,lane,fastq_1,fastq_2
# P1,S1,L001,S1_R1.fq.gz,S1_R2.fq.gz
# P1,S2,L001,S2_R1.fq.gz,S2_R2.fq.gz

# --- nf-core/rnaseq ---
nextflow run nf-core/rnaseq -r 3.14.0 \
   --input samples.csv --outdir rnaseq_results/ \
   --genome GRCh38 --aligner star_salmon \
   -profile singularity

# --- 在 SLURM 上跑 ---
nextflow run nf-core/sarek -r 3.8.1 \
   --input samplesheet.csv --outdir results/ \
   -profile singularity,slurm \
   --max_memory '128.GB' --max_cpus 32

# --- 在 AWS Batch 上跑 ---
nextflow run nf-core/sarek -r 3.8.1 \
   --input s3://my-bucket/samplesheet.csv \
   --outdir s3://my-bucket/results/ \
   -profile awsbatch \
   -bucket-dir s3://my-bucket/work/ \
   -resume
// --- nextflow.config (HPC + Singularity + 資源管控) ---
profiles {
  slurm {
    process {
      executor      = 'slurm'
      queue         = 'normal'
      clusterOptions = '--account=mylab --partition=cpu'

      withLabel: 'process_low'    { cpus = 2;  memory = '8 GB';   time = '2h' }
      withLabel: 'process_medium' { cpus = 8;  memory = '32 GB';  time = '8h' }
      withLabel: 'process_high'   { cpus = 16; memory = '128 GB'; time = '24h' }
    }
    singularity {
      enabled   = true
      autoMounts = true
      cacheDir   = '/scratch/singularity_cache'
    }
  }

  awsbatch {
    process.executor  = 'awsbatch'
    process.queue     = 'my-batch-queue'
    aws.region        = 'us-east-1'
    aws.client.maxConnections = 20
  }
}

// 資源 retry 策略：失敗的 process 自動升級 memory
process {
  errorStrategy = { task.exitStatus in [137,143] ? 'retry' : 'finish' }
  maxRetries    = 3
  memory        = { 8.GB * task.attempt }
}
