定序技術與 FASTQ 格式

從 Sanger 到 Nanopore，理解每個讀長背後的化學原理與資料格式。

From Sanger to Nanopore — understand the chemistry and data formats behind every read.

概覽

為什麼要先理解定序技術？

所有下游分析（變異呼叫、表達量化、結構變異偵測）的 準確度上限 都受限於原始 reads 的特性：讀長、錯誤率、錯誤模式、覆蓋均勻度。短讀長（Illumina）擅長 SNV，但對長重複序列束手無策；長讀長（Nanopore/PacBio）能完整跨越複雜區域，但 base-level 錯誤率較高。選對平台 比選對軟體還重要。

FASTQ 格式則是所有 NGS 分析的「共同語言」——每四行記錄一條 read，第四行的 Phred 品質字元字元背後對應的是 機率而非單一分數。理解這一點對 QC、變異過濾、覆蓋深度估算都至關重要。

The accuracy of every downstream analysis (variant calling, expression quantification, SV detection) is fundamentally bounded by raw read properties: length, error rate, error mode, and coverage uniformity. Short reads (Illumina) excel at SNVs but stumble on long repeats; long reads (Nanopore/PacBio) span complex regions but carry higher base-level errors. Choosing the right platform matters more than choosing the right software.

FASTQ is the lingua franca of NGS — four lines per read, with the 4th-line quality string encoding a probability, not a single score. Understanding this is critical for QC, variant filtering, and depth estimation.

💡
核心原則：讀長 × 通量 × 錯誤率 形成三角取捨。Illumina = 短/高/低；ONT = 長/中/中；PacBio HiFi = 中長/中/極低。 Core principle: Read length × throughput × error rate form an iron triangle. Illumina = short/high/low; ONT = long/medium/medium; PacBio HiFi = medium-long/medium/ultra-low.
平台比較

一、主流定序平台

💎

Illumina (SBS — Sequencing by Synthesis)

當前最普及的短讀長平台（NovaSeq X / NextSeq）。讀長：50–300 bp paired-end。錯誤率：< 0.1%（per base）。強項：單鹼基準確、單次 run 通量極高（NovaSeq X 可達 16 Tb）、單位成本最低。弱項：長重複序列、結構變異、單體型 (haplotype) 解析能力有限。

The dominant short-read platform (NovaSeq X / NextSeq). Read length: 50–300 bp paired-end. Error rate: < 0.1% per base. Strengths: per-base accuracy, massive throughput (up to 16 Tb on NovaSeq X), lowest cost-per-base. Weaknesses: long repeats, structural variants, haplotype phasing.

🧵

Oxford Nanopore (ONT)

讀長最長（單條 read 可達 4 Mb 以上！）的奈米孔技術，鹼基通過孔洞時擾動電流訊號被即時偵測。讀長：中位數 10–100 kb，可達 Mb 級。錯誤率：當前 R10.4.1 化學 < 1%（HAC/SUP basecalling）。強項：即時定序、可攜（MinION）、能直接偵測甲基化（5mC、6mA）。弱項：homopolymer indel 仍是難題。

Longest-read platform — single reads can exceed 4 Mb. Bases are detected as they perturb a current passing through a protein nanopore. Read length: median 10–100 kb, up to Mb-scale. Error rate: <1% with R10.4.1 chemistry (HAC/SUP basecalling). Strengths: real-time sequencing, portable (MinION), direct methylation detection (5mC, 6mA). Weaknesses: homopolymer indels remain challenging.

🌀

PacBio HiFi (Circular Consensus)

SMRT cell 上的 zero-mode waveguide 即時偵測單分子聚合反應；HiFi 模式將同一段 DNA 來回讀多次取共識序列。讀長：15–25 kb。錯誤率：HiFi < 0.1%（與 Illumina 相當！）。強項：長 + 準，是 T2T 完整人類基因組組裝的關鍵；單體型 phasing 出色。弱項：單位成本仍高於 Illumina。

SMRT cells use zero-mode waveguides to detect single-polymerase activity in real time. HiFi mode reads the same DNA fragment in a circle to generate consensus. Read length: 15–25 kb. Error rate: <0.1% — comparable to Illumina! Strengths: long + accurate, key to the T2T human genome; excellent haplotype phasing. Weaknesses: higher per-base cost than Illumina.

📜

Sanger (Capillary)

第一代定序（1977）。如今主要用於 variant 驗證、microsatellite 分析與低通量檢測。讀長：800–1000 bp。錯誤率：< 0.001%。適用：單一片段、低通量、極高準確度。NGS 流程中的「黃金標準」驗證工具。

First-generation sequencing (1977). Today mainly used for variant validation, microsatellite typing, and low-throughput assays. Read length: 800–1000 bp. Error rate: < 0.001%. Use case: single fragments, low throughput, gold-standard validation in NGS workflows.

資料格式

二、FASTQ 格式解剖

每條 read 在 FASTQ 檔中佔用 四行：

Each read occupies four lines in a FASTQ file:

# Line 1: 標頭 (@ 開頭) — read ID、instrument、lane、tile、x/y、flag
@SRR123456.1 HISEQ-1:123:H7Y3HBBXX:2:1101:1456:2086 1:N:0:ATCACG
# Line 2: 鹼基序列 (A/T/G/C/N)
GATCACAGGTCTATCACCCTATTAACCACTCACGGGAGCTCTCCATGCATTTGGTATTTTCG
# Line 3: 分隔符 (+, 可選擇性重複 ID)
+
# Line 4: 品質字串 (ASCII-33 編碼 Phred score)
CCCFFFFFHHHHHJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJJ
⚠️
編碼陷阱：現代資料（Illumina 1.8+、ONT、PacBio）都使用 Phred+33（ASCII 從 ! 開始）。但舊版 Illumina（1.3–1.5）用 Phred+64（@ 開始）。如果 FastQC 報告品質「過高」（出現 J 以上字元）或「過低」，可能是編碼判斷錯誤！ Encoding pitfall: Modern data (Illumina 1.8+, ONT, PacBio) all use Phred+33 (ASCII starts at !). But old Illumina (1.3–1.5) used Phred+64 (starts at @). If FastQC reports "abnormally high" or "low" quality, encoding may be misidentified.

平台選擇

三、如何挑選合適的定序平台？

🌳 平台選擇決策樹

Q1:
你主要分析的是 SNV/Indel，且樣本數大？→ 是 → Illumina NovaSeq X（30× WGS 約 $200，性價比最高）
Q2:
需要解析 結構變異、長重複、HLA 區域？→ 是 → PacBio HiFi（準確的長讀長），或 ONT（更長但需更深覆蓋）
Q3:
要做 de novo 組裝（無參考基因組的物種）？→ 是 → PacBio HiFi + ONT ultra-long + Hi-C（黃金組合）
Q4:
需要 直接偵測甲基化 而不想做 bisulfite？→ 是 → ONT（5mC、6mA 原生偵測）或 PacBio kinetics
Q5:
要 現場即時定序（病原體爆發、田野研究）？→ 是 → ONT MinION（手掌大小、USB 供電）
Q1:
Primarily calling SNV/Indel with many samples? → Yes → Illumina NovaSeq X (30× WGS ~$200, best cost-effectiveness)
Q2:
Need to resolve SVs, long repeats, HLA region? → Yes → PacBio HiFi (accurate long reads), or ONT (longer but needs deeper coverage)
Q3:
Doing de novo assembly (species without reference)? → Yes → PacBio HiFi + ONT ultra-long + Hi-C (gold combination)
Q4:
Want native methylation detection without bisulfite? → Yes → ONT (native 5mC, 6mA) or PacBio kinetics
Q5:
Need real-time field sequencing (outbreaks, field studies)? → Yes → ONT MinION (palm-sized, USB-powered)
🚀
2025-2026 趨勢：越來越多臨床/研究流程採用 「Illumina + 長讀長」混合策略——以 Illumina 提供基礎覆蓋，搭配 30× PacBio HiFi 補上 SV 與 phasing。Pangenome（HPRC Release 2，2025-05）也已釋出 200+ 個基於這種策略組裝的 phased haplotypes。 2025-2026 trend: Many clinical/research workflows now use a hybrid "Illumina + long-read" strategy — Illumina provides baseline coverage, 30× PacBio HiFi adds SV and phasing. 程式碼

FASTQ 操作實作

# 計算 FASTQ 中 read 數量 / Count reads in FASTQ
zcat sample_R1.fastq.gz | wc -l | awk '{print $1/4}'

# 預覽前 2 條 reads / Preview first 2 reads
zcat sample_R1.fastq.gz | head -n 8

# 計算平均讀長 / Average read length
zcat sample_R1.fastq.gz | awk 'NR%4==2 {sum+=length($0); n++} END {print sum/n}'

# 偵測編碼版本（Phred+33 vs +64）/ Detect encoding
zcat sample_R1.fastq.gz | head -n 4000 | awk 'NR%4==0' | fold -w1 | \
  awk '{n=length; for(i=1;i<=n;i++) {c=substr($0,i,1); v=int(c); if(v<min || min==0) min=v; if(v>max) max=v}} END {print min,max}'
# min < 59 → Phred+33; min >= 64 → Phred+64

# 配對檢查（R1/R2 順序一致）/ Pair sanity check
paste <(zcat R1.fq.gz | awk 'NR%4==1') <(zcat R2.fq.gz | awk 'NR%4==1') | head
# 使用 Biopython / Using Biopython
from Bio import SeqIO
import gzip

# 讀取並計算 Q30 比例 / Compute %Q30
total, q30 = 0, 0
with gzip.open("sample_R1.fastq.gz", "rt") as f:
    for rec in SeqIO.parse(f, "fastq"):
        q = rec.letter_annotations["phred_quality"]
        total += len(q)
        q30   += sum(1 for v in q if v >= 30)
print(f"%Q30 = {100*q30/total:.2f}%")

# 高效率串流（不解析）/ Streaming without full parse
with gzip.open("sample_R1.fastq.gz", "rt") as f:
    for i, line in enumerate(f):
        if i % 4 == 1: # sequence line
            print(len(line.strip()))
        if i > 40: break
# ShortRead (Bioconductor)
library(ShortRead)
fq <- readFastq("sample_R1.fastq.gz")
length(fq)                   # read count
mean(width(fq))             # mean read length

# 品質分數矩陣 / Quality score matrix
qm <- as(quality(fq), "matrix")
summary(rowMeans(qm))      # per-read mean Q

# Q30 比例 / %Q30
mean(qm >= 30, na.rm = TRUE)
