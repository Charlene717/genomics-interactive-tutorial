STEP 4 / 14

比對後處理

排序、標記重複、品質校正——讓 BAM 變成「乾淨」的變異呼叫輸入。

Sort, mark duplicates, recalibrate — turn raw BAM into clean input for variant calling.

概覽

為什麼比對後不能直接呼叫變異？

剛從 aligner 出來的 BAM 充滿「假訊號」：PCR duplicates 重複計入同一條 DNA 分子、定序儀的系統性 quality miscalibration、Indel 附近的軟對齊錯誤。不處理這些，HaplotypeCaller 會被假 evidence 騙到 false positive 暴增。

GATK Best Practices 把比對後處理視為「四個必做步驟」：Sort → MarkDuplicates → BQSR → Index。其中 BQSR（Base Quality Score Recalibration）對 SNV 精確度影響特別大；DRAGEN/DeepVariant 在 2025 後逐漸把這一步「內建」進演算法。

Fresh BAM from an aligner carries fake signal: PCR duplicates double-count the same DNA molecule, systematic miscalibration of base qualities, soft-alignment errors near indels. Without cleanup, HaplotypeCaller is tricked by spurious evidence — false positives spike.

GATK Best Practices defines four mandatory post-alignment steps: Sort → MarkDuplicates → BQSR → Index. BQSR especially boosts SNV precision; DRAGEN/DeepVariant in 2025 increasingly "absorb" this step into the caller.

💡
核心原則：順序很重要！ Sort → Mark Duplicates → BQSR。BQSR 必須在 dup-mark 後做，否則 PCR duplicate 的偏誤會污染校正模型。 Core principle: Order matters! Sort → Mark Duplicates → BQSR. BQSR must come after duplicate marking, otherwise PCR duplicates contaminate the recalibration model.
核心步驟

一、四個必做步驟

📚

1. Sort by coordinate

按染色體位置排序 BAM。幾乎所有下游工具（MarkDuplicates、HaplotypeCaller、bcftools）都要求 coordinate-sorted BAM。aligner 輸出本來是「read order」，必須先 sort。
工具：samtools sort -@ 8 或 aligner pipe 直接接 sort。

Sort BAM by chromosome position. Almost every downstream tool (MarkDuplicates, HaplotypeCaller, bcftools) requires coordinate-sorted BAM. Aligner output is in read order; sort first.
Tools: samtools sort -@ 8, or pipe directly from aligner.

🪞

2. Mark Duplicates

PCR duplicate：同一個 DNA 分子被 PCR 放大多次後重複定序，造成假 evidence 累積。MarkDuplicates 不刪除而是標記（FLAG 1024），variant caller 自動忽略。
工具：gatk MarkDuplicates 或更快的 gatk MarkDuplicatesSpark、samtools markdup。

PCR duplicates: the same molecule amplified and sequenced multiple times, inflating fake evidence. MarkDuplicates flags them (FLAG 1024) rather than removing — variant callers ignore flagged reads.
Tools: gatk MarkDuplicates, faster MarkDuplicatesSpark, or samtools markdup.

📏

3. BQSR (Base Quality Score Recalibration)

定序儀回報的 Phred Q 常有系統性偏誤（某些 cycle、某些 context 高估或低估）。BQSR 利用「已知 variants」（dbSNP、Mills indel）建立 covariate 模型重新校正分數。對 SNV 精確度提升明顯，特別是低覆蓋深度時。
工具：gatk BaseRecalibrator + ApplyBQSR。

Reported Phred Q has systematic bias (some cycles/contexts over- or under-estimate). BQSR builds a covariate model using known variants (dbSNP, Mills indels) and recalibrates. Major precision boost for SNVs, especially at low coverage.
Tools: gatk BaseRecalibrator + ApplyBQSR.

🗂️

4. Index

產生 .bai 索引讓 random access（IGV 視覺化、bcftools region query）成為可能。下游沒索引就無法 fetch 區域。
工具：samtools index 或 gatk BuildBamIndex。

Generate .bai index for random access (IGV viewing, bcftools region queries). No index = no region fetch downstream.
Tools: samtools index or gatk BuildBamIndex.

二、Duplicate Rate 與覆蓋的關係

PCR duplicate 比例會隨「定序深度 ÷ library complexity」上升。當你深度測序但 library 複雜度不足，duplicate rate 會飆升而「有效深度」(unique reads) 反而停滯。拖動下方滑桿觀察。

進階

三、UMI 與分子定量

傳統 dedup 只看 位置 + 鏈方向，無法區分「真正同一分子被多次測」與「兩個獨立分子恰好被剪在同一處」。UMI (Unique Molecular Identifier) 在 library prep 階段就為每個分子貼上唯一條碼，讓 dedup 變得真正可靠。

應用場景：cell-free DNA（ctDNA 低 VAF 變異）、scRNA-seq（10X Genomics）、低 input 樣本、qcRNA-seq（quasi count 校正）。工具：UMI-tools、fgbio、UMI-Reducer。

Traditional dedup looks only at position + strand, so it can't distinguish "same molecule re-sequenced" from "two independent molecules happening to share a breakpoint". UMIs (Unique Molecular Identifiers) tag each molecule during library prep, making dedup truly reliable.

Use cases: cell-free DNA (low-VAF ctDNA variants), scRNA-seq (10X Genomics), low-input samples, quasi-count correction. Tools: UMI-tools, fgbio, UMI-Reducer.

決策引導

四、後處理策略決策

🌳 後處理策略

Q1:
WGS / WES 要做 germline variant calling？→ 是 → 完整 GATK 4 步：Sort → MarkDup → BQSR → Index
Q2:
使用 DeepVariant？→ 是 → 仍需 MarkDup，但不需要 BQSR（DeepVariant 內建 quality recalibration）。
Q3:
RNA-seq？→ 是 → 不要 MarkDup！高表現基因會被誤判 duplicate；計數類 DE 通常跳過 dedup。例外：RNA 變異呼叫流程需要 MarkDup 但要小心。
Q4:
ChIP-seq？→ 是 → 移除 duplicates（peak 計數常因 dedup 而更乾淨）；ATAC-seq 也是。
Q5:
樣本有 UMI？→ 是 → 用 UMI-aware dedup（umi_tools dedup / fgbio CallMolecularConsensusReads），效果遠優於位置 dedup。
Q1:
WGS / WES germline variant calling? → Yes → Full GATK 4 steps: Sort → MarkDup → BQSR → Index
Q2:
Using DeepVariant? → Yes → Still mark duplicates, but skip BQSR (DeepVariant has internal quality recalibration).
Q3:
RNA-seq? → Yes → Do not MarkDup! Highly expressed genes will be flagged as duplicates; standard DE pipelines skip dedup. Exception: RNA variant calling needs MarkDup but proceed carefully.
Q4:
ChIP-seq? → Yes → Remove duplicates (cleans peak counts). Same for ATAC-seq.
Q5:
UMI-tagged samples? → Yes → Use UMI-aware dedup (umi_tools dedup / fgbio CallMolecularConsensusReads) — vastly better than position dedup.
⚠️
BQSR 不要做的場景：① 非人類且沒有可靠的 known-sites VCF；② 已用 DeepVariant；③ 比對深度極低（< 5×）；④ amplicon-seq（強烈 enrichment 會誤導校正模型）。 Skip BQSR when: ① Non-human without reliable known-sites VCF; ② Using DeepVariant; ③ Very low coverage (< 5×); ④ Amplicon-seq (strong enrichment misleads recalibration).
程式碼

完整後處理流程

# --- GATK 4 完整後處理 ---
# 1. Mark Duplicates
gatk MarkDuplicates \
    -I S1.sorted.bam \
    -O S1.dedup.bam \
    -M S1.dup.metrics.txt \
    --VALIDATION_STRINGENCY LENIENT

# 2. BQSR — 建立 recalibration table
gatk BaseRecalibrator \
    -I S1.dedup.bam \
    -R ref/hg38.fa \
    --known-sites ref/dbsnp.vcf.gz \
    --known-sites ref/Mills_indels.vcf.gz \
    -O S1.recal.table

# 3. ApplyBQSR — 套用校正
gatk ApplyBQSR \
    -I S1.dedup.bam \
    -R ref/hg38.fa \
    --bqsr-recal-file S1.recal.table \
    -O S1.recal.bam

# 4. Index
samtools index S1.recal.bam

# 5. 檢查（QC after BQSR）
gatk AnalyzeCovariates \
    -bqsr S1.recal.table \
    -plots S1.recal.pdf
# --- samtools 路線（無 BQSR）---
samtools sort -@ 16 -o S1.sorted.bam S1.bam
samtools index S1.sorted.bam

samtools markdup -@ 16 -r S1.sorted.bam S1.dedup.bam     # -r 真正移除（非標記）
samtools index S1.dedup.bam

samtools flagstat S1.dedup.bam
samtools stats   S1.dedup.bam | grep ^SN | head -20

# --- MarkDuplicatesSpark（快 2–4×）---
gatk MarkDuplicatesSpark \
    -I S1.sorted.bam \
    -O S1.dedup.bam \
    --conf 'spark.executor.cores=16'

# --- UMI-aware dedup（fgbio）---
fgbio GroupReadsByUmi -i S1.bam -o S1.grouped.bam -s adjacency
fgbio CallMolecularConsensusReads -i S1.grouped.bam -o S1.consensus.bam -M 1
// 完整後處理（含 BQSR）/ Full post-processing with BQSR
workflow POSTALIGN {
  take: sorted_bam_ch
        known_sites_ch
        fasta_ch

  main:
    MARKDUP(sorted_bam_ch)
    BQSR_TABLE(MARKDUP.out.bam, fasta_ch, known_sites_ch)
    APPLY_BQSR(MARKDUP.out.bam, BQSR_TABLE.out.table, fasta_ch)

  emit:
    recal_bam = APPLY_BQSR.out.bam
}

process MARKDUP {
  input:  tuple val(meta), path(bam)
  output: tuple val(meta), path("*.dedup.bam"), path("*.bai"), emit: bam
          path "*.metrics", emit: metrics
  script:
  """
  gatk MarkDuplicates -I ${bam} -O ${meta.id}.dedup.bam -M ${meta.id}.metrics
  samtools index ${meta.id}.dedup.bam
  """
}
