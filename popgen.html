群體遺傳學

在群體尺度理解基因頻率、結構、選擇與演化歷史。

Allele frequencies, structure, selection, and demographic history at the population scale.

概覽

群體遺傳學在問什麼？

群體遺傳學的核心問題：「為什麼這個族群的基因頻率是這個樣子？」——背後是四個力量：突變、漂變、遷移、選擇。從族群 VCF 出發，我們可以：

  • 用 PCA / ADMIXTURE 描述族群結構（誰跟誰近、有沒有 admixture）
  • 用 Fst 量化族群間分化
  • 用 Tajima's D、π、θ_W 估計變異與選擇
  • 用 iHS、XP-EHH、PBS 偵測自然選擇
  • 用 PSMC、SMC++、SLiM 推斷有效族群大小（Ne）與分裂歷史

這些方法既是基礎科學，也是 GWAS / 醫學基因體學的必要前置（控制族群結構）。

The central question of population genetics: "Why are allele frequencies in this population the way they are?" — driven by four forces: mutation, drift, migration, selection. From a cohort VCF you can:

  • Use PCA / ADMIXTURE to describe population structure (who is close to whom, any admixture?)
  • Use Fst to quantify between-population differentiation
  • Use Tajima's D, π, θ_W for diversity and selection signatures
  • Use iHS, XP-EHH, PBS for selection scans
  • Use PSMC, SMC++, SLiM to infer effective population size (Ne) and split history

These methods are both basic science and a necessary precondition for GWAS / medical genomics (controlling for population structure).

💡
核心原則：分析前一定要做 PCA。歐裔 + 東亞裔 + 非裔 pool 在一起跑 GWAS，PC1 會解釋 80% 變異而不是疾病訊號——這叫 population stratification。 Core principle: Always run PCA first. Pooling European + East Asian + African samples in a single GWAS makes PC1 explain 80% of variance via ancestry instead of phenotype — that's population stratification.
核心指標

一、必懂的群體遺傳量

π — 核苷酸多樣性

任兩條序列在某位點的平均差異機率。意義：族群內遺傳多樣性。典型：人類 ≈ 0.001（每 1 kb 約 1 個差異）；果蠅 ≈ 0.01；某些自交植物 ≈ 0.0001。低 π 可能代表 bottleneck 或近交。

Average probability of two sequences differing at a site. Meaning: within-population diversity. Typical: humans ≈ 0.001 (~1 diff per kb); fruit fly ≈ 0.01; selfing plants ≈ 0.0001. Low π may indicate bottlenecks or inbreeding.

Tajima's D

比較 π 與 Watterson's θ_W（基於 SNP 數）。意義：D ≈ 0 中性；D < 0 暗示族群擴張或 positive selection（過多 rare alleles）；D > 0 暗示 balancing selection 或 bottleneck（中等頻率 allele 過多）。

Compares π with Watterson's θ_W (SNP-count-based). D ≈ 0: neutral; D < 0: population expansion or positive selection (excess of rare alleles); D > 0: balancing selection or bottleneck (excess of intermediate-frequency alleles).

Fst — 族群分化

量化兩族群間的等位基因頻率差異。0 = 完全相同、1 = 完全不同。人類洲際 Fst ≈ 0.05–0.15；歐洲內部 ≈ 0.005–0.01；雙胞胎物種間 0.5+。Weir & Cockerham 1984 是經典 estimator。

Quantifies allele-frequency differentiation between populations. 0 = identical, 1 = completely separated. Human continental Fst ≈ 0.05–0.15; within-Europe ≈ 0.005–0.01; sister species 0.5+. Weir & Cockerham 1984 is the classical estimator.

iHS / XP-EHH — 選擇訊號

偵測近期 positive selection。iHS（族群內）：核心 allele 周圍 haplotype 長度比較；最近的 selective sweep 會留下長 high-frequency haplotype。XP-EHH（跨族群）：比較同一位點在兩族群的 haplotype 長度；單一族群選擇會留下不對稱訊號。

Detect recent positive selection. iHS (within-population): compares haplotype length around a core allele; recent selective sweeps leave long high-frequency haplotypes. XP-EHH (cross-population): compares haplotype lengths between two populations; single-population selection leaves an asymmetric signal.

四、典型分析流程

🌳 從 VCF 到群體洞見

1.
QC：MAF ≥ 0.05、missingness、HWE、移除親屬（KING）
2.
LD 修剪：PLINK --indep-pairwise 50 5 0.2 取得 ~100K independent SNPs
3.
PCA：plink2 --pca 10 → 散佈圖 (PC1 vs PC2)；尋找 outlier 與群集
4.
ADMIXTURE：跑 K = 2…10，用 CV error 找最佳 K，繪 stacked bar plot
5.
分化：scikit-allel 計算 pairwise Fst；找 Fst 異常高的位點（可能選擇）
6.
選擇掃描：iHS（族群內）、XP-EHH（跨族群）、PBS（三族群）
7.
歷史：SMC++ 或 IBDNe 推估 Ne(t) 曲線
1.
QC: MAF ≥ 0.05, missingness, HWE, remove relatives (KING)
2.
LD pruning: PLINK --indep-pairwise 50 5 0.2 → ~100K independent SNPs
3.
PCA: plink2 --pca 10 → scatter (PC1 vs PC2); identify outliers and clusters
4.
ADMIXTURE: run K = 2…10, use CV error to pick best K; plot stacked bar
5.
Differentiation: scikit-allel for pairwise Fst; flag outlier sites (possible selection)
6.
Selection scans: iHS (within-pop), XP-EHH (cross-pop), PBS (three-pop)
7.
Demographic history: SMC++ or IBDNe to infer Ne(t) trajectory
程式碼

實作範例

# --- PCA with PLINK 2.0 ---
plink2 --vcf cohort.vcf.gz \
   --maf 0.05 --geno 0.05 --hwe 1e-6 \
   --indep-pairwise 50 5 0.2 --out prune
plink2 --vcf cohort.vcf.gz --extract prune.prune.in --make-bed --out cohort.pruned

plink2 --bfile cohort.pruned --pca 10 --out pca
# pca.eigenvec: FID IID PC1 PC2 ... PC10

# --- ADMIXTURE for K=2..6 with cross-validation ---
for K in 2 3 4 5 6; do
   admixture --cv cohort.pruned.bed $K | tee log${K}.txt
done
grep -h CV log*.txt
# 選 CV error 最低的 K

# --- pairwise Fst ---
plink2 --bfile cohort.pruned \
   --pheno populations.tsv --pheno-name POP \
   --fst POP --out fst_results
# scikit-allel 完整群體遺傳分析
import allel, numpy as np

callset = allel.read_vcf("cohort.vcf.gz")
gt = allel.GenotypeArray(callset["calldata/GT"])
samples = callset["samples"]
pop = np.array(["EUR","EAS","AFR"][i%3] for i in range(len(samples)))

# π (nucleotide diversity) per population
for p in ["EUR","EAS","AFR"]:
    idx = np.where(pop == p)[0]
    sub = gt.take(idx, axis=1)
    ac  = sub.count_alleles()
    pi  = allel.sequence_diversity(callset["variants/POS"], ac)
    print(p, "π =", pi)

# Tajima's D in windows
ac = gt.count_alleles()
td = allel.windowed_tajima_d(
        callset["variants/POS"], ac, size=100_000)
print("Tajima D windows:", td[0].mean())

# Pairwise Fst (EUR vs AFR)
eur_idx = np.where(pop == "EUR")[0]
afr_idx = np.where(pop == "AFR")[0]
ac_eur = gt.take(eur_idx, axis=1).count_alleles()
ac_afr = gt.take(afr_idx, axis=1).count_alleles()
num, den = allel.hudson_fst(ac_eur, ac_afr)
fst = np.sum(num) / np.sum(den)
print("Hudson Fst (EUR vs AFR) =", fst)

# PCA
gn = gt.to_n_alt()
coords, model = allel.pca(gn, n_components=10, scaler="patterson")
print("Variance explained:", model.explained_variance_ratio_[:3])
# --- R: SNPRelate ---
library(SNPRelate)

# 把 VCF 轉成 GDS
snpgdsVCF2GDS("cohort.vcf.gz", "cohort.gds", method="biallelic.only")
gds <- snpgdsOpen("cohort.gds")

# LD prune
snpset <- snpgdsLDpruning(gds, ld.threshold=0.2, slide.max.bp=50000)
prune_snps <- unlist(snpset)

# PCA
pca <- snpgdsPCA(gds, snp.id=prune_snps, num.thread=8)
df  <- data.frame(sample=pca$sample.id, PC1=pca$eigenvect[,1],
                  PC2=pca$eigenvect[,2], pop=pheno$pop)
library(ggplot2)
ggplot(df, aes(PC1, PC2, color=pop)) + geom_point(alpha=0.7)

# IBD / kinship
ibd <- snpgdsIBDKING(gds, snp.id=prune_snps, num.thread=8)
kin <- snpgdsIBDSelection(ibd, kinship.cutoff=0.0884)
head(kin)

# 計算 Fst
fst <- snpgdsFst(gds, population=as.factor(pheno$pop), method="W&C84")
cat("Overall Fst:", fst$Fst, "\n")
