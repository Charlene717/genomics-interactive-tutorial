STEP 6 / 14

變異註釋與過濾

把 VCF 裡的位點轉成「這個變異是什麼、會做什麼、有多稀有」。

Turn VCF coordinates into "what this variant is, what it does, and how rare it is".

概覽

從位置到生物學意義

VCF 給的是 chr1:155235855 G>A——但臨床或研究最關心的是：它落在哪個基因？造成 missense / stop-gain / synonymous？是 ClinVar 已知 pathogenic？在 gnomAD 多稀有？

變異註釋工具吃 VCF 吐強化 VCF（多加 INFO 欄位），把基因位置、protein change、人口頻率、ClinVar 致病性、in-silico 預測（CADD、REVEL、AlphaMissense）都打上去。然後再做變異過濾——保留稀有、有功能影響的位點。

A VCF gives you chr1:155235855 G>A — but clinical/research interpretation needs: which gene? missense / stop-gain / synonymous? Is it ClinVar pathogenic? How rare in gnomAD?

Annotation tools consume VCF and emit enriched VCF (more INFO fields) tagged with gene location, protein change, population frequencies, ClinVar pathogenicity, in-silico predictions (CADD, REVEL, AlphaMissense). Then filter to keep rare, functional candidates.

💡
核心原則：單一註釋工具不足以下臨床判斷。跨工具一致 (cross-tool consensus) 才是黃金標準——2025 ClinVar 研究顯示 VEP、SnpEff、ANNOVAR 對 LoF 分類存在顯著差異。 Core principle: A single annotator is insufficient for clinical decisions. Cross-tool consensus is the gold standard — a 2025 ClinVar study found significant disagreement between VEP, SnpEff, and ANNOVAR on LoF classifications.
主流工具

一、三大主流註釋器

🐎

VEP (Ensembl)

Ensembl 維護的 perl-based 註釋器。強項：transcript 覆蓋最廣（67,695 transcripts）、HGVSp 準確度最高 (0.977)、官方 API 與 plugin 系統（CADD、REVEL、AlphaMissense、SpliceAI 都有 plugin）。適用：研究、臨床、雲端規模。

Ensembl's perl-based annotator. Strengths: widest transcript coverage (67,695), highest HGVSp accuracy (0.977), rich plugin system (CADD, REVEL, AlphaMissense, SpliceAI). Use: research, clinical, cloud scale.

🐍

SnpEff

Java-based，自帶大量物種預建 database。強項：HGVSc 表現最佳 (0.988)、支援 SV 註釋、容易整合進 pipeline、執行速度快。適用：非模式物種、SV 註釋、teaching。

Java-based, ships with prebuilt databases for many species. Strengths: best HGVSc accuracy (0.988), supports SV annotation, easy pipeline integration, fast. Use: non-model species, SV annotation, teaching.

📊

ANNOVAR

perl-based，注重表格化輸出。強項：輸出 TSV 易讀、整合 dbNSFP（含 50+ 預測分數）、適合下游 R/Python 分析。弱項：商業授權限制、transcript 庫更新較慢。

Perl-based, focused on tabular output. Strengths: TSV-friendly output, integrates dbNSFP (50+ prediction scores), easy R/Python downstream. Weakness: commercial license restrictions, slower transcript database updates.

人口資料庫

四、必用的人口頻率資料庫

🌐 gnomAD v4 (2024)

當前最大規模整合：807K exomes + 76K genomes，跨 8 大祖先群體。核心欄位： AF (整體)、AF_popmax (各群體最高)、AC_hom (同型 alt 個數)。原則：真正致病的罕見變異通常 AF_popmax < 0.001 且 AC_hom = 0（或極少）。

Currently the largest integration: 807K exomes + 76K genomes across 8 ancestry groups. Key fields: AF (overall), AF_popmax (max across populations), AC_hom (homozygous alt count). Principle: truly pathogenic rare variants typically have AF_popmax < 0.001 and AC_hom near zero.

📝 ClinVar (NCBI)

已知致病性註解資料庫。分級：Pathogenic / Likely-Pathogenic / VUS（uncertain）/ Likely-Benign / Benign。注意：同一變異常有多個 submitter 結論衝突，需看 review status (★ 數越多越可信)。

Curated database of known pathogenicity. Tiers: Pathogenic / Likely-Pathogenic / VUS (uncertain) / Likely-Benign / Benign. Caveat: the same variant often has conflicting submitter calls — check review status (more ★ = more reliable).

🔮 dbNSFP / CADD / AlphaMissense

dbNSFP 聚合 SIFT、PolyPhen2、REVEL 等 40+ 預測分數。CADD 整合多 feature 為單一分數（≥ 20 = top 1% 有害）。AlphaMissense（DeepMind, 2023）用 AlphaFold 結構預測 missense 致病性，2024 後成為新一代主流分數。

dbNSFP aggregates SIFT, PolyPhen2, REVEL, and 40+ scores. CADD integrates many features into a single score (≥ 20 = top 1% deleterious). AlphaMissense (DeepMind, 2023) uses AlphaFold structure to predict missense pathogenicity — a new standard since 2024.

✂️ SpliceAI

Illumina 開發的 deep-learning splice predictor。預測一個位點是否影響 splice donor/acceptor。常用閾值：delta score ≥ 0.2（敏感）、≥ 0.5（嚴格）。適用：VEP 的 deep splice 機制；非編碼變異功能解讀關鍵。

Illumina's deep-learning splice predictor. Outputs whether a variant disrupts splice donor/acceptor. Thresholds: delta score ≥ 0.2 (sensitive), ≥ 0.5 (strict). Use: deep splice interpretation in VEP; essential for non-coding variant interpretation.

程式碼

註釋與過濾實作

# --- Ensembl VEP (Docker) ---
vep --cache --offline --dir_cache vep_cache/ \
    --assembly GRCh38 --species homo_sapiens \
    --input_file cohort.vcf.gz --output_file cohort.vep.vcf.gz \
    --vcf --compress_output bgzip \
    --canonical --symbol --hgvs --pubmed \
    --plugin AlphaMissense,file=AlphaMissense_hg38.tsv.gz \
    --plugin CADD,/refs/whole_genome_SNVs.tsv.gz \
    --plugin SpliceAI,snv=/refs/spliceai_scores.snv.hg38.vcf.gz \
    --custom /refs/gnomad.v4.exomes.vcf.bgz,gnomADe,vcf,exact,0,AF,AF_popmax \
    --custom /refs/clinvar.vcf.gz,ClinVar,vcf,exact,0,CLNSIG,CLNDN \
    --fork 8

# --- SnpEff (Java, 物種多) ---
snpEff -v GRCh38.105 cohort.vcf.gz > cohort.snpeff.vcf
snpEff stats -csvStats stats.csv cohort.snpeff.vcf

# --- ANNOVAR (TSV 友善) ---
table_annovar.pl cohort.vcf humandb/ -buildver hg38 \
  -out cohort -remove \
  -protocol refGene,clinvar_20240611,gnomad41_exome,dbnsfp47a \
  -operation g,f,f,f -nastring . -vcfinput
# --- bcftools 多層過濾 ---
# 1) caller PASS
bcftools view -f PASS cohort.vep.vcf.gz -Oz -o pass.vcf.gz

# 2) gnomAD AF < 0.01
bcftools view -i 'INFO/gnomADe_AF_popmax<0.01 | INFO/gnomADe_AF_popmax="."' pass.vcf.gz \
  -Oz -o rare.vcf.gz

# 3) 編碼區 + splice 功能性後果 (用 VEP 的 CSQ 欄位)
bcftools +split-vep rare.vcf.gz -c Consequence -O u | \
  bcftools view -i 'INFO/Consequence~"missense|stop_gained|frameshift|splice_acceptor|splice_donor"' \
  -Oz -o coding.vcf.gz

# 4) CADD ≥ 20
bcftools view -i 'INFO/CADD_PHRED>=20' coding.vcf.gz -Oz -o candidates.vcf.gz
bcftools index candidates.vcf.gz
# cyvcf2 + pandas 走訪 VEP CSQ
from cyvcf2 import VCF
import pandas as pd

vcf = VCF("cohort.vep.vcf.gz")
csq_keys = vcf.get_header_type("CSQ")["Description"].split("Format: ")[1].split("|")

rows = []
for v in vcf:
    if v.FILTER: continue
    csq_list = v.INFO.get("CSQ", "").split(",")
    for entry in csq_list:
        d = dict(zip(csq_keys, entry.split("|")))
        if d.get("PICK") != "1": continue     # canonical only
        rows.append({
            "chrom": v.CHROM, "pos": v.POS,
            "gene": d.get("SYMBOL"),
            "consequence": d.get("Consequence"),
            "hgvsp": d.get("HGVSp"),
            "af_popmax": float(d.get("gnomADe_AF_popmax") or 0),
            "cadd": float(d.get("CADD_PHRED") or 0),
            "clinvar": d.get("ClinVar_CLNSIG")
        })

df = pd.DataFrame(rows)
# 篩 candidate
cands = df[(df.af_popmax < 0.001) & (df.cadd >= 20) &
           (df.consequence.str.contains("missense|stop_gained|frameshift|splice"))]
print(cands.head())
