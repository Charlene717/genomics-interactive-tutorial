DNA 甲基化與表觀基因體

5mC 在 CpG 上的調控密碼——WGBS、RRBS、與長讀長原生偵測。

The 5mC regulatory code at CpGs — WGBS, RRBS, and long-read native detection.

概覽

為什麼甲基化重要？

DNA 甲基化（哺乳類主要為 CpG dinucleotide 上的 5mC）是穩定的表觀調控標記。啟動子 CpG island 高甲基化通常壓抑基因表現；基因體區 (gene body) 甲基化則常與活躍轉錄正相關。差異甲基化區域 (DMR) 在腫瘤、發育、老化、神經退化都有重要角色。

偵測甲基化的兩條路：(1) Bisulfite sequencing——非甲基化 C 被亞硫酸氫鈉轉成 U（測得 T），甲基化 C 保持為 C；比較讀出來的 C/T 比例。(2) 長讀長原生偵測——Nanopore 與 PacBio HiFi 能直接從電流/動力學訊號區分 5mC，無需 bisulfite。

DNA methylation (mainly 5mC at CpG dinucleotides in mammals) is a stable epigenetic mark. Promoter CpG island hypermethylation typically silences genes; gene-body methylation often correlates with active transcription. Differentially methylated regions (DMRs) play roles in cancer, development, aging, and neurodegeneration.

Two detection routes: (1) Bisulfite sequencing — unmethylated C is converted to U (sequenced as T) by sodium bisulfite, methylated C stays as C; compare the C/T ratio. (2) Native long-read detection — Nanopore and PacBio HiFi distinguish 5mC directly from current/kinetics signals, no bisulfite needed.

💡
核心原則：Bisulfite 化學會把 ref 中的所有 C → T（除了 5mC 的 C），因此標準 aligner 無法直接用，必須用 Bisulfite-aware aligner（Bismark、bwa-meth）。CpH (CHG、CHH) 在植物與胚胎幹細胞重要。 Core principle: Bisulfite converts all unmethylated C → T (except 5mC), so standard aligners cannot map directly — use bisulfite-aware aligners (Bismark, bwa-meth). CpH context (CHG, CHH) matters in plants and embryonic stem cells.

二、Beta value & M value 轉換

甲基化分數（methylation fraction, β）= methylated reads / total reads。β 介於 0–1。雖然直觀，但不適合線性模型（heteroscedastic：β 接近 0 或 1 時變異被壓縮）。M value = log2(β / (1-β)) 是常見的線性化轉換，更適合 DE 分析。拖滑桿觀察兩者關係。

Methylation fraction β = methylated reads / total reads, in [0, 1]. Intuitive but not suitable for linear models — heteroscedastic, variance is compressed near 0 and 1. M value = log2(β / (1-β)) is the standard linearizing transform for DE-like analyses. Drag the slider.

3D Genome

三、Hi-C 與 3D 基因體

Hi-C 用 crosslink + restriction digestion + ligation 捕捉空間鄰近的 DNA 對，定序產生 contact matrix，揭示 3D 染色體結構：

  • Compartment A/B：~Mb 級別的「活躍 / 抑制」分區（兩者甲基化模式差異大）。
  • TAD (Topologically Associating Domain)：數百 kb–1 Mb，邊界由 CTCF + cohesin 限定，TAD 內接觸頻率高。
  • Loops / Enhancer-Promoter contacts：kb–百 kb 的精細接觸，常對應功能調控。

工具：HiC-Pro / Juicer / cooler / hicexplorer。應用：整合 Hi-C + ChIP-seq + ATAC-seq + Methylation 重建細胞型別的 epigenome landscape。

Hi-C uses crosslink + restriction digest + ligation to capture spatially adjacent DNA pairs, sequenced into a contact matrix revealing 3D structure:

  • A/B compartments: ~Mb-scale "active/repressed" partitions (their methylation profiles differ markedly).
  • TADs (Topologically Associating Domains): few hundred kb–1 Mb, bounded by CTCF + cohesin; within-TAD contacts are enriched.
  • Loops / Enhancer-Promoter contacts: kb to hundreds of kb fine-scale contacts, often functional.

Tools: HiC-Pro / Juicer / cooler / hicexplorer. Use: integrate Hi-C + ChIP-seq + ATAC-seq + methylation to reconstruct a cell-type epigenome landscape.

四、技術選擇與分析陷阱

🌳 甲基化技術選擇

Q1:
大規模流行病學 / EWAS（n > 1000）？→ 是 → EPIC array（成本、可比性最佳）
Q2:
focus 於 CGI / promoter 的小型研究？→ 是 → RRBS（性價比佳）
Q3:
需要全基因體解析（如腫瘤完整 methylome）？→ 是 → WGBS 或 EM-seq（後者 library 品質更穩）
Q4:
需要 phasing（allele-specific methylation）或不想做 bisulfite？→ 是 → Nanopore native 或 PacBio HiFi + 5mC basecaller
Q5:
差異分析？→ 是 → methylKit（基因組級 logistic）或 DSS（beta-binomial）；EPIC 用 limma + M-values
Q1:
Large-scale epidemiology / EWAS (n > 1000)? → Yes → EPIC array (cost, comparability)
Q2:
Focused study on CGI / promoters? → Yes → RRBS (cost-effective)
Q3:
Need whole-genome methylome (e.g., comprehensive tumor)? → Yes → WGBS or EM-seq (latter more stable library)
Q4:
Need phasing (allele-specific methylation) or avoid bisulfite? → Yes → Nanopore native or PacBio HiFi + 5mC basecaller
Q5:
Differential analysis? → Yes → methylKit (genome-wide logistic) or DSS (beta-binomial); for EPIC use limma + M-values
⚠️
常見錯誤：① 把 bisulfite reads 餵給 BWA → 大量未比對；② 沒過濾 SNP（C→T SNP 會偽裝成 demethylation）；③ 對 β value 直接做 t-test（heteroscedasticity 違反假設）；④ 沒考慮細胞型別組成（whole-blood methylation 主要受白血球比例影響——必要時用 RefBase / EpiDISH 校正）。 Common errors: ① Feeding bisulfite reads to BWA → massive unmapped; ② Not filtering SNPs (a C→T SNP masquerades as demethylation); ③ T-test directly on β-values (violates homoscedasticity); ④ Ignoring cell-type composition (whole-blood methylation is dominated by leukocyte proportions — correct with RefBase / EpiDISH when needed).
程式碼

甲基化分析實作

# --- Bismark (WGBS / RRBS) ---
# 1) 建 bisulfite genome index（一次性）
bismark_genome_preparation ref/

# 2) 比對 (paired-end)
bismark --genome ref/ \
   -1 trim_R1.fq.gz -2 trim_R2.fq.gz \
   -o bismark_out/ \
   --multicore 4

# 3) Dedup（WGBS 必做；RRBS 不需要）
deduplicate_bismark --paired bismark_out/sample_R1_bismark_bt2_pe.bam

# 4) 抽取每 CpG 的 methylation call
bismark_methylation_extractor --paired-end --comprehensive --cytosine_report \
   --genome_folder ref/ \
   -o methcalls/ \
   bismark_out/sample.deduplicated.bam

# 結果在 .CX_report.txt（CpG/CHG/CHH 個別列出）
# --- Nanopore native 5mC（modkit + dorado） ---
# 1) basecalling 含 5mC modifications
dorado basecaller --modified-bases 5mCG_5hmCG \
   model_path/ \
   pod5_dir/ > reads.bam

# 2) 比對
samtools fastq -T MM,ML reads.bam | \
  minimap2 -y -ax map-ont -t 16 ref/hg38.fa - | \
  samtools sort -@ 8 -o ont.sorted.bam -
samtools index ont.sorted.bam

# 3) modkit 抽 per-CpG methylation
modkit pileup ont.sorted.bam pileup.bed.gz \
   --ref ref/hg38.fa \
   --cpg --combine-strands \
   --threads 16

# pileup.bed.gz: chrom start end mod_code N_valid_cov frac_modified ...
# --- R: methylKit (WGBS/RRBS DMR) ---
library(methylKit)

files <- list("S1.bismark.cov.gz", "S2.bismark.cov.gz",
              "S3.bismark.cov.gz", "S4.bismark.cov.gz")
myObj <- methRead(files,
   sample.id   = list("S1","S2","S3","S4"),
   assembly    = "hg38",
   pipeline    = "bismarkCoverage",
   treatment   = c(0,0,1,1),
   context     = "CpG",
   mincoverage = 10
)

meth  <- unite(myObj, destrand=FALSE)
diff  <- calculateDiffMeth(meth, mc.cores=4)
hyper <- getMethylDiff(diff, difference=25, qvalue=0.01, type="hyper")
hypo  <- getMethylDiff(diff, difference=25, qvalue=0.01, type="hypo")

# --- R: minfi (EPIC array) ---
library(minfi); library(limma)
rgSet  <- read.metharray.exp("idat_dir/")
mSet   <- preprocessQuantile(rgSet)
M      <- getM(mSet)        # M-values 用於 limma
design <- model.matrix(~ pheno$condition + pheno$age + pheno$sex)
