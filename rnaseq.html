概覽

Bulk RNA-seq DE 在做什麼？

Bulk RNA-seq 把組織中所有細胞的 mRNA 混合定序，得到每個基因（或 transcript）的「平均表現量」。差異表達分析（DE）試圖回答：「在我感興趣的條件對比下（treated vs control，腫瘤 vs 正常），哪些基因表達顯著改變？」

流程：FASTQ → QC → Alignment (STAR/HISAT2) 或 quasi-mapping (Salmon) → 計數矩陣 (counts × samples) → 過濾低表現基因 → normalization → 統計模型 (DESeq2/edgeR/limma-voom) → 多重檢驗校正 → 視覺化 (Volcano、MA、heatmap、PCA)。

Bulk RNA-seq sequences pooled mRNA from a tissue, yielding "average expression" per gene/transcript. DE analysis asks: "Under my contrast of interest (treated vs control, tumor vs normal), which genes change significantly?"

Pipeline: FASTQ → QC → Alignment (STAR/HISAT2) or quasi-mapping (Salmon) → counts matrix (genes × samples) → filter low-expression genes → normalization → statistical model (DESeq2/edgeR/limma-voom) → multiple-testing correction → visualization (volcano, MA, heatmap, PCA).

💡
核心原則：不要用 raw counts 直接比樣本之間；不要用 TPM/FPKM 做 DE。用 DESeq2/edgeR/limma-voom 的標準流程——它們同時處理 size factor、dispersion、與基因內變異。 Core principle: Never compare raw counts directly between samples; never use TPM/FPKM for DE. Use the standard DESeq2/edgeR/limma-voom workflow — they handle size factors, dispersion, and within-gene variation together.
核心步驟

一、五步分析流程

1. Mapping / Quantification

兩條路：(a) splice-aware alignment（STAR / HISAT2 → BAM → featureCounts/HTSeq）拿到 gene-level counts；(b) quasi-mapping（Salmon / kallisto）直接拿 transcript-level abundance。Salmon 速度快 10–100×，並輸出 effective length 與 inferential uncertainty，搭配 tximport 可直接餵 DESeq2。

Two routes: (a) splice-aware alignment (STAR / HISAT2 → BAM → featureCounts/HTSeq) for gene-level counts; (b) quasi-mapping (Salmon / kallisto) for transcript-level abundance. Salmon is 10–100× faster and outputs effective length + inferential uncertainty — feed into DESeq2 via tximport.

2. Filter low-expression

移除「至少 N 個樣本中 CPM ≥ 1」的基因以下者。低計數基因統計力差且讓多重檢驗校正更嚴格，剔除後 power 提升。常見：edgeR filterByExpr()。

Remove genes failing "CPM ≥ 1 in at least N samples". Low-count genes have poor power and tighten multiple-testing correction; removing them increases power. Common: edgeR filterByExpr().

3. Normalization

校正樣本間 library size 與 composition bias。DESeq2 用 size factor（median-of-ratios）；edgeR 用 TMM；limma-voom 用 TMM + voom transformation 把 counts 轉成 log-CPM 並估計 mean-variance trend。

Adjust between-sample library size and composition bias. DESeq2 uses size factors (median-of-ratios); edgeR uses TMM; limma-voom uses TMM + voom transformation to log-CPM with mean-variance trend.

4. Modeling & Testing

用 negative binomial（DESeq2、edgeR）或 linear model on log-CPM with weights（limma-voom）擬合每基因。Wald test 或 likelihood ratio test 比較感興趣的 contrast。務必用 design matrix 控制 batch / sex / age 等混雜因子。

Fit negative binomial (DESeq2, edgeR) or linear model on log-CPM with weights (limma-voom) per gene. Wald test or likelihood ratio test compares the contrast of interest. Always include batch/sex/age in the design matrix to control confounders.

5. Multiple testing & Visualization

對 ~15K–20K 基因檢驗會有大量假陽性。用 Benjamini-Hochberg FDR 校正（DESeq2 預設）；常見 padj < 0.05 + |log2FC| ≥ 1 過濾。視覺化：volcano、MA、heatmap of top genes、PCA、sample-to-sample distance heatmap。

Testing 15K–20K genes generates many false positives. Apply Benjamini-Hochberg FDR (DESeq2 default); common threshold padj < 0.05 + |log2FC| ≥ 1. Visualize: volcano, MA, top-gene heatmap, PCA, sample-to-sample distance heatmap.

三、DESeq2 vs edgeR vs limma-voom

Tool模型速度最適情境
DESeq2NB GLM, Wald★★★小樣本 (n < 10)，最穩健Small samples (n < 10), most robust
edgeR (glmQLFit)NB GLM, QL F-test★★★多因子、時序、cohortMulti-factor, time-course, cohorts
limma-voomLinear on log-CPM★★★★★大樣本 (n > 20)、複雜 designLarge samples (n > 20), complex designs
📊
2025 共識：在標準 bulk RNA-seq（≥ 3 replicates）情境下三者結果 > 90% 重疊。選哪個更多是團隊偏好而非客觀勝負。除非有特殊理由，照團隊既有 pipeline 走最穩。 2025 consensus: For standard bulk RNA-seq (≥ 3 replicates), all three agree on > 90% of DE genes. Choose what your team already uses — switching brings no objective gain.

四、設計與分析陷阱

🌳 實驗設計檢查清單

1.
n ≥ 3 per group，更好 n ≥ 5。低重複會嚴重低估 dispersion。
2.
避免 confounded batch：不要把 condition 全部跑同一 batch；把 control 與 treated 在每 batch 平均分配。
3.
用 PCA 檢查批次效應。若 PC1 對應 batch 而非 condition，立刻警覺；在 design 加 batch term。
4.
strand-specific library 要對應計數工具參數（-s 1/2 for featureCounts；--libtype ISR/ISF for Salmon）。
5.
不要做 duplicate removal（RNA 中 dup 多半是真實高表現）。
1.
n ≥ 3 per group, ideally n ≥ 5. Low replication severely underestimates dispersion.
2.
Avoid confounded batches: don't run all condition X in one batch; spread control and treated across batches.
3.
Check batch effects with PCA. If PC1 captures batch instead of condition, beware — add batch term in design.
4.
Match strand-specific library to counting tool params (-s 1/2 for featureCounts; --libtype ISR/ISF for Salmon).
5.
Do not remove duplicates (most RNA duplicates are real high expression).
程式碼

完整 RNA-seq DE 流程

# --- Salmon (quasi-mapping) ---
# 1) 建立 decoy-aware index（一次性）/ Build decoy-aware index (one-time)
grep "^>" ref/hg38.fa | awk '{print substr($1,2)}' > decoys.txt
cat ref/gencode.v45.transcripts.fa ref/hg38.fa > gentrome.fa
salmon index -t gentrome.fa -d decoys.txt -i salmon_idx -k 31

# 2) 每樣本 quasi-mapping
for s in S1 S2 S3 S4; do
  salmon quant -i salmon_idx -l A \
    -1 trim_${s}_R1.fq.gz -2 trim_${s}_R2.fq.gz \
    --gcBias --validateMappings -p 8 \
    -o quants/${s}
done

# 結果在 quants/${s}/quant.sf — 用 tximport 進 R / Result in quants/${s}/quant.sf → tximport in R
# --- R: tximport + DESeq2 ---
library(tximport); library(DESeq2)
library(AnnotationDbi); library(org.Hs.eg.db)

# 1) 樣本表 / Sample sheet
samples <- data.frame(
  sample    = c("S1","S2","S3","S4"),
  condition = factor(c("ctrl","ctrl","trt","trt")),
  batch     = factor(c("b1","b2","b1","b2"))
)

# 2) tximport — 把 transcript counts 摺到 gene-level
files <- file.path("quants", samples$sample, "quant.sf")
tx2gene <- read.csv("tx2gene.csv")
txi <- tximport(files, type="salmon", tx2gene=tx2gene)

# 3) DESeq2 dataset, 帶 batch 校正
dds <- DESeqDataSetFromTximport(txi, samples, ~ batch + condition)
dds <- DESeq(dds)

# 4) filter 低表現
dds <- dds[rowSums(counts(dds)) >= 10, ]

# 5) 結果與校正
res <- results(dds, contrast=c("condition","trt","ctrl"), alpha=0.05)
res <- lfcShrink(dds, coef="condition_trt_vs_ctrl", type="apeglm")
summary(res)

# 6) 視覺化 / Visualization
vsd <- vst(dds, blind=FALSE)
plotPCA(vsd, intgroup = "condition")
DESeq2::plotMA(res, ylim=c(-5,5))

# 7) Volcano (用 EnhancedVolcano)
library(EnhancedVolcano)
EnhancedVolcano(res, lab=rownames(res),
   x="log2FoldChange", y="padj",
   pCutoff=0.05, FCcutoff=1)
# --- edgeR (glmQLFit) ---
library(edgeR)
y    <- DGEList(counts=count_matrix, group=samples$condition)
keep <- filterByExpr(y, design=model.matrix(~ samples$batch + samples$condition))
y    <- y[keep, , keep.lib.sizes=FALSE]
y    <- calcNormFactors(y, method="TMM")
design <- model.matrix(~ samples$batch + samples$condition)
y    <- estimateDisp(y, design, robust=TRUE)
fit  <- glmQLFit(y, design, robust=TRUE)
qlf  <- glmQLFTest(fit, coef="samples$conditiontrt")
topTags(qlf, n=20)

# --- limma-voom (大樣本 / 複雜 design) ---
library(limma)
v   <- voom(y, design, plot=TRUE)
fit <- lmFit(v, design)
fit <- eBayes(fit)
topTable(fit, coef="samples$conditiontrt", n=20)
// --- nf-core/rnaseq (一行跑完所有流程) ---
nextflow run nf-core/rnaseq -r 3.14.0 \
   --input samplesheet.csv \
   --outdir results/ \
   --genome GRCh38 \
   --aligner star_salmon \
   --pseudo_aligner salmon \
   --extra_salmon_quant_args '--gcBias' \
   -profile docker
