GWAS 全基因組關聯分析

在百萬 SNP 上找與性狀關聯的位點——QC、群體分層、imputation、Manhattan plot。

Find SNPs associated with traits across millions of variants — QC, stratification, imputation, Manhattan plots.

概覽

GWAS 在做什麼？

GWAS 在數十萬到百萬位點上分別檢驗：「該位點的基因型與感興趣性狀（疾病、身高、BMI）是否關聯？」。輸出的顯著位點不代表致病基因，而是「關聯訊號的座標」——通常落在非編碼區，需要 fine-mapping 與功能驗證才能找到 causal variant。

核心流程：基因型 (chip 或 WGS) → QC（樣本 + SNP）→ Imputation → 群體分層校正（PCA）→ 關聯模型（線性/邏輯回歸或 mixed model）→ 多重檢驗校正（5 × 10⁻⁸）→ Manhattan plot + fine-mapping → 功能解讀（eQTL、enhancer 重疊）。

GWAS tests millions of variants independently: "Is genotype at this site associated with the trait of interest (disease, height, BMI)?". Significant hits are not causal genes but coordinates of an association signal — usually non-coding, requiring fine-mapping and functional validation.

Pipeline: genotypes (chip or WGS) → QC (samples + SNPs) → imputation → population structure correction (PCA) → association model (linear / logistic / mixed model) → multiple testing correction (5 × 10⁻⁸) → Manhattan plot + fine-mapping → functional interpretation (eQTL, enhancer overlap).

💡
核心原則：顯著閾值 p < 5 × 10⁻⁸ 是「Bonferroni for ~1M independent tests」的慣例。GWAS hit 多半位於非編碼區——它是 座標，不是基因；fine-mapping (SuSiE, FINEMAP) 才告訴你 causal variant。 Core principle: Genome-wide significance p < 5 × 10⁻⁸ is "Bonferroni for ~1M independent tests". GWAS hits are mostly non-coding — they are coordinates, not genes; fine-mapping (SuSiE, FINEMAP) reveals the causal variant.
QC 步驟

一、GWAS QC 必做清單

📋 SNP-level QC

• call rate ≥ 0.95（missingness ≤ 5%）
• MAF ≥ 0.01（common variants；rare variant 需另套方法）
• HWE p ≥ 10⁻⁶（control 中的嚴重 HWE 偏離 = 基因型錯誤訊號）
• 移除 strand-ambiguous SNPs（A/T、C/G）若要做 meta-analysis

• Call rate ≥ 0.95 (missingness ≤ 5%)
• MAF ≥ 0.01 (common variants; rare variants need different methods)
• HWE p ≥ 10⁻⁶ in controls (severe deviation suggests genotyping errors)
• Remove strand-ambiguous SNPs (A/T, C/G) for meta-analysis

👥 Sample-level QC

• call rate ≥ 0.97
• 性別檢查（X 染色體 heterozygosity）
• 移除親屬（KING / PLINK --king-cutoff 0.0884，約 2nd-degree relatives）
• 異質性離群（excess heterozygosity = 樣本污染或汙染）
• ancestry outlier（與 1000G 做 PCA 找 outlier）

• Call rate ≥ 0.97
• Sex check (X-chromosome heterozygosity)
• Remove relatives (KING / PLINK --king-cutoff 0.0884, ~2nd-degree)
• Heterozygosity outliers (excess = contamination)
• Ancestry outliers (PCA with 1000G to flag outliers)

二、Manhattan Plot 模擬

Manhattan plot 把 p-value 取 -log10 後沿染色體繪出。GWAS 顯著閾值 5 × 10⁻⁸ ⇒ -log10(p) ≈ 7.3。拖滑桿調整效應量 (effect size) 與樣本數，觀察 power 變化。

Manhattan plot displays -log10(p) along chromosomes. Genome-wide significance 5 × 10⁻⁸ ⇒ -log10(p) ≈ 7.3. Adjust effect size and sample size to see power changes.

進階主題

四、Imputation、PRS 與 Mendelian Randomization

🔮 Genotype Imputation

用 reference panel（TOPMed、HRC、1000G）填補晶片未覆蓋的 SNP。邏輯：LD 區段的 phased haplotype 可從 reference 推斷。工具：Michigan Imputation Server、TOPMed Imputation Server（線上免費）；本地用 Minimac4 / IMPUTE5。產出：~50M imputed SNPs（每位點附 INFO score；INFO ≥ 0.7 通常保留）。

Use a reference panel (TOPMed, HRC, 1000G) to fill in SNPs not on the chip. Logic: phased haplotypes within LD blocks can be inferred from the reference. Tools: Michigan Imputation Server, TOPMed Imputation Server (online, free); locally Minimac4 / IMPUTE5. Output: ~50M imputed SNPs with INFO scores (typically keep INFO ≥ 0.7).

🎯 Polygenic Risk Score (PRS)

把 GWAS summary statistics 在新個體上加權求和，得到該性狀的「遺傳負擔」分數。方法：P+T (clumping + threshold)、PRS-CS (continuous shrinkage)、LDpred2、SBayesR。注意：跨族裔 portability 仍是大問題——歐裔訓練的 PRS 在非歐裔的解釋力會降 50–70%。

Weighted sum of GWAS summary stats applied to a new individual, yielding a "genetic burden" score. Methods: P+T (clumping + threshold), PRS-CS, LDpred2, SBayesR. Caveat: cross-ancestry portability remains a major issue — PRS trained on European populations loses 50–70% explanatory power in non-European groups.

🧪 Mendelian Randomization (MR)

把 GWAS 顯著 SNP 當「natural randomization」工具變數，推斷暴露 → 疾病的因果效應。主流方法：IVW、MR-Egger、Weighted Median、MR-PRESSO（移除 pleiotropic outlier）。工具：TwoSampleMR (R)、MendelianRandomization。

Treats GWAS-significant SNPs as "natural randomization" instrument variables to infer causal effects of exposure → disease. Methods: IVW, MR-Egger, Weighted Median, MR-PRESSO (removes pleiotropic outliers). Tools: TwoSampleMR (R), MendelianRandomization.

🔗 Colocalization & QTL

判斷 GWAS 訊號是否與 eQTL / sQTL 共享因果變異。工具：coloc、SuSiE-coloc、HyPrColoc。典型用法：用 GTEx v9 (838 donors, 54 tissues, 2024) 中對應組織的 eQTL 與 GWAS 比對。

Tests whether a GWAS signal shares a causal variant with eQTL / sQTL. Tools: coloc, SuSiE-coloc, HyPrColoc. Typical use: compare against GTEx v9 (838 donors, 54 tissues, 2024) eQTLs from matching tissue.

Q1:
n < 10K, no relatives? → Yes → PLINK 2.0 --glm (most direct)
Q2:
n > 50K quantitative trait (BMI, height)? → Yes → REGENIE or BOLT-LMM
Q3:
Highly unbalanced case-control (1:100+) or rare disease? → Yes → SAIGE (saddle point approximation mandatory)
Q4:
Cohort with relatives or cryptic relatedness? → Yes → SAIGE or BOLT-LMM (mixed model)
Q5:
Have GWAS hits, want fine-mapping? → Yes → SuSiE (R) or FINEMAP → coloc → MR
⚠️
陷阱：① λ_GC > 1.1 警示嚴重 population stratification — 重新做 PCA + 加更多 PC；② 不要把多個族裔 pool 在同一 GWAS — 用 within-ancestry GWAS + meta-analysis (METAL)；③ 不要把 GWAS hit 直接當「致病基因」報導 — 多數是非編碼，需要 fine-mapping & functional 驗證。 Pitfalls: ① λ_GC > 1.1 signals serious population stratification — redo PCA + add more PCs; ② Don't pool multiple ancestries in one GWAS — use within-ancestry GWAS + meta-analysis (METAL); ③ Don't report GWAS hits as "causal genes" — most are non-coding and require fine-mapping + functional validation.
程式碼

GWAS 完整流程

# --- PLINK 2.0 完整 QC + GWAS ---
# 1) SNP-level QC
plink2 --bfile raw \
   --maf 0.01 --geno 0.05 --hwe 1e-6 \
   --make-bed --out qc1

# 2) Sample-level QC
plink2 --bfile qc1 \
   --mind 0.03 --king-cutoff 0.0884 \
   --make-bed --out qc2

# 3) PCA（先 LD prune）
plink2 --bfile qc2 --indep-pairwise 50 5 0.2 --out prune
plink2 --bfile qc2 --extract prune.prune.in --pca 10 --out pca

# 4) 關聯分析（含 PC 校正）
plink2 --bfile qc2 \
   --pheno phenotype.txt --pheno-name disease \
   --covar pca.eigenvec --covar-name PC1-PC10,age,sex \
   --glm hide-covar \
   --out gwas

# 5) 看顯著位點
awk '$13 < 5e-8' gwas.disease.glm.logistic | head
# --- REGENIE (UK Biobank-scale) ---
# Step 1: whole-genome ridge regression on a LD-pruned subset
regenie \
   --step 1 \
   --bed qc2 \
   --extract prune.prune.in \
   --phenoFile pheno.txt --phenoCol disease \
   --covarFile covar.txt --covarColList PC1-PC10,age,sex \
   --bsize 1000 \
   --bt --out step1

# Step 2: SNP-by-SNP test using step 1 predictions
regenie \
   --step 2 \
   --bgen imputed.bgen --sample imputed.sample \
   --phenoFile pheno.txt --phenoCol disease \
   --covarFile covar.txt --covarColList PC1-PC10,age,sex \
   --bt --firth --approx \
   --pred step1_pred.list \
   --bsize 400 \
   --out gwas

# Meta-analysis with METAL
metal << EOF

MARKER SNP
WEIGHT N
ALLELE A1 A2
EFFECT BETA
STDERR SE
PVAL P
PROCESS study1.txt
PROCESS study2.txt
ANALYZE HETEROGENEITY
EOF
