STEP 3 / 14

序列比對 (Read Alignment)

把每一條 read 放回基因組正確位置——理解 BWA、STAR、Minimap2 各自的最佳場景。

Place each read back on the reference — understand when to use BWA, STAR, or Minimap2.

概覽

為什麼比對是 NGS 的核心？

從 FASTQ 進入有意義的生物學，必須先回答一個問題：「這條 read 來自基因組哪個位置？」。比對演算法面對的挑戰是：基因組有 3 Gb，每個 read 30 億選一；而且 read 可能有 sequencing error、SNP、indel，甚至跨越 splice junction（RNA-seq）。

下游的所有 metric（mapping quality、coverage、variant evidence、splice junction、SV breakpoint）都建立在比對結果之上。選錯比對工具或參數，後面再厲害的軟體也救不回來。

To extract biology from FASTQ, you must answer: "Where in the genome did this read come from?" The challenge: 3 Gb genome, billion-to-one position selection, with reads carrying sequencing errors, SNPs, indels, even splice junctions (RNA-seq).

Every downstream metric (mapping quality, coverage, variant evidence, splice junctions, SV breakpoints) builds on alignment. Wrong aligner or parameters = no software downstream can save you.

💡
核心原則：DNA → BWA-MEM2 (短) 或 Minimap2 (長)；RNA → STAR 或 HISAT2 (考慮 splice)；偽比對 (pseudo-alignment) → Salmon (RNA 計數最快)。 Core principle: DNA → BWA-MEM2 (short) or Minimap2 (long); RNA → STAR or HISAT2 (splice-aware); pseudo-alignment → Salmon (fastest RNA quantification).
工具比較

一、主流比對工具

🎯

BWA-MEM2

短讀長 DNA 比對的事實標準。BWT-FM index + Maximal Exact Match (MEM) 演算法，產生與 BWA-MEM 完全相同的結果，但速度快 1.3–3× 且記憶體更省。2025 主流 pipeline（nf-core/sarek、GATK best practices）預設使用。

De facto standard for short-read DNA alignment. BWT-FM index + Maximal Exact Match (MEM); produces identical results to BWA-MEM but 1.3–3× faster with smaller memory footprint. Default in 2025 mainstream pipelines (nf-core/sarek, GATK best practices).

⭐

STAR

RNA-seq 短讀長首選。splice-aware——使用 suffix array 同時尋找 splice junction。強項：準確、可輸出 chimeric reads (fusion detection)、可 2-pass 模式提升新 splice 偵測。弱項：記憶體消耗大（人類 ~30 GB）。

Top choice for short-read RNA-seq. Splice-aware via suffix array. Strengths: accurate, outputs chimeric reads (fusion detection), 2-pass mode improves novel junction discovery. Weakness: high memory (~30 GB for human).

🚀

HISAT2

另一個 splice-aware RNA-seq 比對器，使用 Graph FM index 與 hierarchical FM。記憶體效率高（~8 GB），雖然準確度比 STAR 略低，仍是低資源環境的好選擇；也支援 DNA 比對。

Alternative splice-aware aligner using Graph FM index + hierarchical FM. Memory efficient (~8 GB); accuracy slightly below STAR but excellent for resource-limited setups; also supports DNA.

🏹

Minimap2

長讀長（ONT、PacBio）比對王者。minimizer-based seed + chaining 演算法；同一工具支援 DNA、RNA、assembly-to-genome、long-read overlap。當前 ONT 與 PacBio 官方推薦。也能比 short-read，但通常 BWA-MEM2 更快。

King of long-read alignment (ONT, PacBio). Minimizer seed + chaining; single tool supports DNA, RNA, assembly-to-genome, long-read overlap. Officially recommended by ONT and PacBio. Can align short reads but BWA-MEM2 is usually faster.

🏁

Bowtie2

輕量、低資源的選擇。FM-index based，對 short reads（≤ 50 bp）特別友善。ChIP-seq / ATAC-seq 仍常見其身影（包含 ENCODE pipeline）。BWA-MEM2 對 ≥ 70 bp 通常更準確。

Lightweight, low-resource choice. FM-index based; especially friendly to short reads (≤ 50 bp). Still common in ChIP-seq / ATAC-seq (including ENCODE pipelines). BWA-MEM2 is usually more accurate for ≥ 70 bp.

⚡

Salmon / kallisto (pseudo-alignment)

RNA-seq 計數的「跳過比對」捷徑。直接將 reads 映射到 transcriptome k-mer index，輸出 transcript-level 計數。速度比 STAR 快 10–100×，但不輸出 BAM；用於下游 DE 分析、不適合 fusion / SV / variant calling。

"Skip alignment" shortcut for RNA-seq quantification. Maps reads directly to transcriptome k-mer index, outputs transcript-level counts. 10–100× faster than STAR; no BAM output. Use for DE; not suited for fusions/SV/variant calling.

互動模擬

二、CIGAR 字串解碼器

CIGAR 描述 read 與 reference 的對齊操作：M=match/mismatch、I=insertion、D=deletion、S=soft clip、N=skip (intron)、H=hard clip。輸入一條 CIGAR 來看視覺化。

CIGAR describes the alignment operations between read and reference: M=match/mismatch, I=insertion, D=deletion, S=soft clip, N=skip (intron), H=hard clip. Enter a CIGAR to visualize.

關鍵概念

三、SAM/BAM 必懂的欄位

FLAG (column 2)

bitwise flags 編碼 read 的狀態。常用：
• 4 = unmapped
• 16 = reverse strand
• 256 = secondary
• 1024 = duplicate
• 2048 = supplementary (split / chimeric)
用 samtools flagstat 一覽全樣本統計。

Bitwise flags encoding read state. Common:
• 4 = unmapped
• 16 = reverse strand
• 256 = secondary
• 1024 = duplicate
• 2048 = supplementary (split / chimeric)
Use samtools flagstat for a sample-wide summary.

MAPQ (column 5)

Phred-scaled mapping quality：該 read 比對位置正確的機率。0 = 多重比對 (multi-mapping)、60 = unique。GATK 預設過濾 MAPQ < 20。請注意各 aligner 對 MAPQ 上限有差（BWA 60, Bowtie2 42, STAR 255）。

Phred-scaled mapping quality: probability the alignment is correct. 0 = multi-mapping, 60 = unique. GATK filters MAPQ < 20 by default. Note: aligners cap MAPQ differently (BWA 60, Bowtie2 42, STAR 255).

TLEN / Insert Size (column 9)

paired-end 兩條 read 間的「外緣」距離。正常 Illumina 短 insert 200–600 bp。異常大 TLEN 暗示可能的 deletion / inversion；異常小或負值 暗示 inverted insert 或 dovetail。

"Outer" distance between paired-end reads. Normal Illumina insert is 200–600 bp. Abnormally large TLEN suggests possible deletions/inversions; tiny or negative values indicate inverted or dovetailed inserts.

NM tag

read 與 reference 的編輯距離（mismatch + indel 鹼基數）。配合 CIGAR 可推算 mismatch 數。SV/variant pipeline 常用此 tag 過濾雜訊比對。

Edit distance between read and reference (mismatches + indel bases). Combined with CIGAR, lets you derive mismatch counts. SV/variant pipelines often filter noisy alignments by this tag.

決策引導

四、比對工具選擇

🌳 比對工具決策樹

Q1:
讀長 > 500 bp（ONT / PacBio）？→ 是 → Minimap2（DNA 用 -ax map-ont 或 -ax map-hifi；RNA 用 -ax splice）
Q2:
RNA-seq 短讀長且需要 BAM（fusion / variant / coverage）？→ 是 → STAR 2-pass
Q3:
RNA-seq 只需要 gene/transcript 計數，且記憶體有限？→ 是 → Salmon（quasi-mapping）
Q4:
DNA / WGS / WES？→ 是 → BWA-MEM2（drop-in 取代 BWA-MEM）
Q5:
ChIP-seq / ATAC-seq 短讀長？→ 是 → Bowtie2 或 BWA-MEM2 都可（ENCODE 標準為 Bowtie2）
Q1:
Read > 500 bp (ONT / PacBio)? → Yes → Minimap2 (DNA: -ax map-ont / -ax map-hifi; RNA: -ax splice)
Q2:
Short-read RNA-seq, need BAM (fusion / variant / coverage)? → Yes → STAR 2-pass
Q3:
RNA-seq, only need gene/transcript counts, limited memory? → Yes → Salmon (quasi-mapping)
Q4:
DNA / WGS / WES? → Yes → BWA-MEM2 (drop-in for BWA-MEM)
Q5:
Short-read ChIP-seq / ATAC-seq? → Yes → Bowtie2 or BWA-MEM2 (ENCODE standard: Bowtie2)
⚠️
常見陷阱：① 把 RNA-seq 直接丟給 BWA-MEM2 → 跨 splice 的 reads 全 soft-clip 丟失；② STAR index 沒帶 --sjdbGTFfile（GTF）→ 失去已知 junction 加速；③ 比對到 Decoy 序列卻沒過濾 unmapped → 下游 coverage 失準。 Common pitfalls: ① Feeding RNA-seq directly to BWA-MEM2 → reads spanning splice junctions are soft-clipped away; ② STAR index without --sjdbGTFfile → loses known-junction acceleration; ③ Aligning to decoy sequences without filtering unmapped → corrupts downstream coverage.
程式碼

比對實作範例

# --- BWA-MEM2 (DNA, paired-end Illumina) ---
# 1. 建立 index（一次性）/ Build index (one-time)
bwa-mem2 index -p ref/hg38 ref/hg38.fa

# 2. 比對 + 直接排序 / Align + sort in one pipe
bwa-mem2 mem -t 16 -R "@RG\tID:S1\tSM:S1\tLB:lib1\tPL:ILLUMINA" \
    ref/hg38 trim_R1.fq.gz trim_R2.fq.gz | \
  samtools sort -@ 8 -o S1.sorted.bam -
samtools index S1.sorted.bam
samtools flagstat S1.sorted.bam

# --- STAR (RNA-seq, 2-pass) ---
STAR --runMode genomeGenerate \
     --genomeDir star_idx \
     --genomeFastaFiles ref/hg38.fa \
     --sjdbGTFfile ref/gencode.v45.gtf \
     --sjdbOverhang 100 --runThreadN 16

STAR --runMode alignReads \
     --genomeDir star_idx \
     --readFilesIn trim_R1.fq.gz trim_R2.fq.gz --readFilesCommand zcat \
     --twopassMode Basic \
     --outSAMtype BAM SortedByCoordinate \
     --outSAMstrandField intronMotif \
     --outFileNamePrefix S1. --runThreadN 16

# --- Minimap2 (long reads) ---
# ONT DNA
minimap2 -ax map-ont -t 16 ref/hg38.fa nanopore.fq.gz | samtools sort -o ont.bam
# PacBio HiFi DNA
minimap2 -ax map-hifi -t 16 ref/hg38.fa hifi.fq.gz | samtools sort -o hifi.bam
# long-read RNA
minimap2 -ax splice -uf -k14 ref/hg38.fa rna.ont.fq | samtools sort -o rna.ont.bam
# 用 pysam 走訪 BAM / Iterate BAM with pysam
import pysam

bam = pysam.AlignmentFile("S1.sorted.bam", "rb")

# 統計 MAPQ 分佈 / MAPQ distribution
from collections import Counter
mapq = Counter()
for read in bam.fetch("chr1", 1000000, 2000000):
    if not read.is_unmapped:
        mapq[read.mapping_quality] += 1
print(sorted(mapq.items()))

# 計算 region 的覆蓋深度 / Per-base coverage
cov = bam.count_coverage("chr1", 1000000, 1000100)
total = [sum(b) for b in zip(*cov)]
print(f"Mean cov: {sum(total)/len(total):.1f}×")

# 篩選 split reads（SV 訊號）/ Filter split reads (SV signal)
for read in bam.fetch():
    if read.has_tag("SA"):     # supplementary alignment
        print(read.query_name, read.reference_start, read.get_tag("SA"))
# Rsamtools / GenomicAlignments
library(Rsamtools)
library(GenomicAlignments)

# 讀取部分 BAM 為 GRanges / Read partial BAM as GRanges
param <- ScanBamParam(
  which = GRanges("chr1", IRanges(1e6, 2e6)),
  what  = c("qname", "flag", "mapq", "cigar", "isize")
)
aln <- readGAlignments("S1.sorted.bam", param = param)
summary(mapq(aln))

# MAPQ 直方圖
hist(mapq(aln), breaks = 50, main = "MAPQ distribution",
     xlab = "MAPQ", col = "steelblue")
// BWA-MEM2 + sort in a single Nextflow process
process BWAMEM2 {
  tag "$meta.id"
  cpus 16; memory '48 GB'

  input:
    tuple val(meta), path(reads)
    path  index
    path  fasta

  output:
    tuple val(meta), path("*.sorted.bam"), path("*.bai"), emit: bam

  script:
  def rg = "@RG\\\\tID:${meta.id}\\\\tSM:${meta.sample}\\\\tLB:${meta.lib}\\\\tPL:ILLUMINA"
  """
  bwa-mem2 mem -t ${task.cpus} -R "${rg}" ${index}/hg38 ${reads[0]} ${reads[1]} \\
    | samtools sort -@ 4 -o ${meta.id}.sorted.bam -
  samtools index ${meta.id}.sorted.bam
  """
}
