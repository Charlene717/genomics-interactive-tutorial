STEP 13 / 14

功能富集分析

從基因清單到生物學故事——GO、KEGG、Reactome、GSEA。

From gene lists to biological narratives — GO, KEGG, Reactome, GSEA.

概覽

從 200 個 DE genes 到生物學意義

DESeq2 給你 200 個 padj < 0.05 的基因——下一步呢？逐個查太累；找它們有沒有「共同功能」就是富集分析（functional enrichment）。

核心問題：「在我的基因清單裡，某個基因集合 (gene set，例如 KEGG 的 'apoptosis pathway') 出現的次數，是不是顯著多於隨機？」。兩大主流方法：

  • ORA（Over-Representation Analysis）：把 DE 基因 vs background 基因做 Fisher's exact test / hypergeometric test。簡單、直觀，但依賴你選的 DE 閾值。
  • GSEA（Gene Set Enrichment Analysis）：用全部基因依 fold change 排序，看 pathway 基因是否聚在排序的兩端。不需設 DE 閾值，對微小但一致的效應更敏感。

DESeq2 hands you 200 genes with padj < 0.05 — now what? Checking each by hand is impossible; finding shared functional themes is what enrichment analysis does.

The core question: "In my gene list, does a gene set (e.g., KEGG's apoptosis pathway) appear more often than random?". Two mainstream approaches:

  • ORA (Over-Representation Analysis): Fisher's exact / hypergeometric test of DE genes vs background. Simple, intuitive, but depends on your DE threshold.
  • GSEA (Gene Set Enrichment Analysis): rank all genes by fold change, ask whether a pathway's genes cluster at either end. No DE threshold needed; more sensitive to small but coherent effects.
💡
核心原則：背景集（universe）的選擇影響結果！要用「實驗中被測試過的所有基因」而非「全基因組」——否則 RNA-seq 中常常表現的家族管理基因會永遠出現。 Core principle: The background set (universe) matters! Use "all genes that were tested in the experiment", not the whole genome — otherwise housekeeping genes that are always expressed in RNA-seq will always show up.
資料庫

一、主流 gene set 資料庫

🌳 GO (Gene Ontology)

三個本體：BP（biological process，如 "apoptosis"）、MF（molecular function，如 "DNA binding"）、CC（cellular component，如 "mitochondrion"）。層級樹狀結構，可以「parent term」聚合 specific term 結果。陷阱：GO term 之間有強烈冗餘——需要做 REVIGO 或 simplifyEnrichment 去冗。

Three ontologies: BP (biological process, e.g., "apoptosis"), MF (molecular function, e.g., "DNA binding"), CC (cellular component, e.g., "mitochondrion"). Hierarchical tree allows parent-term aggregation. Pitfall: Strong redundancy between GO terms — collapse with REVIGO or simplifyEnrichment.

🛤️ KEGG Pathways

手繪 metabolic / signaling pathway map（如 "TGF-β signaling"、"cell cycle"）。適用：分子代謝、信號傳導。缺點：KEGG 路徑通常較小（~50–200 基因）、商業使用授權需付費。

Hand-curated metabolic / signaling pathway maps (e.g., "TGF-β signaling", "cell cycle"). Use: molecular metabolism and signaling. Limit: KEGG pathways are usually small (~50–200 genes); commercial use requires a license.

🌐 Reactome

免費開放、覆蓋更全面的 pathway 資料庫。強項：反應級（reaction-level）的精細註解、跨物種、整合 Ensembl。適用：需要詳細分子過程的研究。

Free, more comprehensive pathway database. Strengths: reaction-level annotation, cross-species coverage, Ensembl integration. Use: when detailed molecular processes matter.

📚 MSigDB (Hallmark, C2, C5...)

Broad Institute 維護，是 GSEA 的「官方」gene set collection。Hallmark (H) 50 個高品質精煉的常用 set；C2 CGP 收集發表論文的 perturbation signature；C7 immunologic signatures；C8 cell-type signatures。

Maintained by the Broad Institute, the "official" gene set collection for GSEA. Hallmark (H): 50 distilled, high-quality sets; C2 CGP: published perturbation signatures; C7: immunologic signatures; C8: cell-type signatures.

GSEA

三、GSEA 原理

GSEA 跑步沿著「依 log2FC 排序的全基因 list」：每遇到 gene set 內的基因加分、外的減分，得到累積 running enrichment score (ES)。最大 ES 偏離 0 的程度作為統計量，與 random permutation 比對得 p-value。輸出包含：

  • NES (Normalized ES)：跨 gene set 比較的標準化 score
  • FDR q-value：多重檢驗校正後的顯著性
  • Leading edge subset：對訊號貢獻最大的基因集合（可用於後續解讀）

選 ranking metric：RNA-seq 常用 log2FC × -log10(padj) 或 signed t-statistic。重要：排序要 有方向（包含正負），不是 absolute fold change。

GSEA walks along the "ranked-by-log2FC gene list": each gene in the set adds, each gene outside subtracts, producing a running enrichment score (ES). The maximum deviation from zero is the statistic, compared against random permutations for p-value. Outputs include:

  • NES (Normalized ES): a standardized score comparable across gene sets
  • FDR q-value: multiple-testing-corrected significance
  • Leading edge subset: genes driving the signal, useful for downstream interpretation

Ranking metric: for RNA-seq use log2FC × -log10(padj) or signed t-statistic. Critical: the ranking must be signed (includes direction), not absolute fold change.

決策引導

四、方法與工具選擇

🌳 ORA vs GSEA

Q1:
有清晰的 DE list（padj < 0.05、|log2FC| > 1）？→ 是 → ORA（clusterProfiler::enrichGO / enrichKEGG / gprofiler2::gost）
Q2:
很多基因有「微小但一致」的方向性改變但不過 DE 閾值？→ 是 → GSEA（clusterProfiler::GSEA / fgsea）
Q3:
非基因清單（如 GWAS SNP）？→ 是 → MAGMA（gene-level then gene-set）或 FUMA
Q4:
ChIP-seq peak？→ 是 → GREAT（將 peak 映射到 nearest gene + regulatory domain）
Q5:
需要視覺化網路？→ 是 → enrichplot（dotplot、emapplot、cnetplot）；Cytoscape EnrichmentMap for publication
Q1:
Clear DE list (padj < 0.05, |log2FC| > 1)? → Yes → ORA (clusterProfiler::enrichGO/enrichKEGG, gprofiler2::gost)
Q2:
Many genes with "small but coherent" directional change, below DE threshold? → Yes → GSEA (clusterProfiler::GSEA, fgsea)
Q3:
Non-gene-list input (e.g., GWAS SNPs)? → Yes → MAGMA (gene-level then gene-set) or FUMA
Q4:
ChIP-seq peaks? → Yes → GREAT (maps peaks to nearest gene + regulatory domain)
Q5:
Need network visualization? → Yes → enrichplot (dotplot, emapplot, cnetplot); Cytoscape EnrichmentMap for publication
⚠️
陷阱：① 用全基因組當 background 而非「測到的基因」→ 偽富集；② GO terms 高度冗餘 → 用 simplifyEnrichment / REVIGO；③ 取 |log2FC| 排序丟給 GSEA → 訊號方向性消失；④ 對 KEGG hsa 編號用未轉換的 Ensembl ID → empty result。 Pitfalls: ① Using whole-genome as background instead of "tested genes" → false enrichments; ② GO terms are highly redundant → use simplifyEnrichment / REVIGO; ③ Feeding |log2FC| (unsigned) to GSEA → loses direction information; ④ Querying KEGG hsa with unconverted Ensembl IDs → empty result.
程式碼

實作範例

# --- R: clusterProfiler ORA (GO + KEGG) ---
library(clusterProfiler)
library(org.Hs.eg.db)
library(enrichplot)

# DE list (Ensembl IDs)
de_genes <- res |> subset(padj < 0.05 & abs(log2FoldChange) > 1) |> rownames()

# background = 所有測到的基因（全部 row）/ background = all tested genes
universe <- rownames(res)

# 轉換 Ensembl → Entrez（KEGG 需要）/ Convert Ensembl → Entrez for KEGG
de_ent  <- bitr(de_genes, fromType="ENSEMBL", toType="ENTREZID", OrgDb=org.Hs.eg.db)
uni_ent <- bitr(universe,  fromType="ENSEMBL", toType="ENTREZID", OrgDb=org.Hs.eg.db)

# GO BP
go <- enrichGO(
  gene          = de_genes,
  universe      = universe,
  OrgDb         = org.Hs.eg.db,
  keyType       = "ENSEMBL",
  ont           = "BP",
  pAdjustMethod = "BH",
  qvalueCutoff  = 0.05,
  readable      = TRUE
)
dotplot(go, showCategory = 15)

# KEGG
kegg <- enrichKEGG(
  gene          = de_ent$ENTREZID,
  universe      = uni_ent$ENTREZID,
  organism      = "hsa",
  pvalueCutoff  = 0.05
)
dotplot(kegg, showCategory = 15)

# 簡化冗餘 GO terms / Collapse redundant GO terms
library(simplifyEnrichment)
go_simplify <- simplify(go, cutoff=0.7, by="p.adjust")
emapplot(pairwise_termsim(go))
# --- R: GSEA via clusterProfiler / fgsea ---
library(clusterProfiler); library(msigdbr)

# 排序：方向性 ranking metric / Signed ranking metric
rank_vec <- res$log2FoldChange * -log10(res$pvalue)
names(rank_vec) <- rownames(res)
rank_vec <- sort(rank_vec[!is.na(rank_vec)], decreasing = TRUE)

# MSigDB Hallmark gene sets
h_sets <- msigdbr(species = "Homo sapiens", category = "H") |>
          dplyr::select(gs_name, ensembl_gene)

gsea <- GSEA(
  geneList  = rank_vec,
  TERM2GENE = h_sets,
  pvalueCutoff = 0.05,
  eps          = 0
)

dotplot(gsea, showCategory = 20, split = ".sign") +
  facet_grid(~ .sign)

# Enrichment plot for a specific set
gseaplot2(gsea, geneSetID = "HALLMARK_TNFA_SIGNALING_VIA_NFKB")

# fgsea (faster) ---
library(fgsea)
fg <- fgsea(pathways = split(h_sets$ensembl_gene, h_sets$gs_name),
            stats    = rank_vec,
            minSize  = 15, maxSize = 500, nproc = 4)
fg[order(padj)][1:15]
# --- Python: gseapy ---
import gseapy as gp
import pandas as pd

# 1) ORA (Enrichr)
de_list = pd.read_csv("de_genes.txt", header=None)[0].tolist()
enr = gp.enrichr(
    gene_list = de_list,
    gene_sets = ["GO_Biological_Process_2023", "KEGG_2021_Human"],
    organism  = "Human",
    outdir    = "enrichr_out"
)
print(enr.results.head())

# 2) Pre-ranked GSEA
rnk = pd.read_csv("rank.tsv", sep="\t", header=None, names=["gene","score"])
gsea = gp.prerank(
    rnk        = rnk,
    gene_sets  = "h.all.v2024.1.Hs.symbols.gmt",
    threads    = 8,
    min_size   = 15,
    max_size   = 500,
    permutation_num = 1000,
    outdir     = "gsea_out"
)
print(gsea.res2d.sort_values("FDR q-val").head())
